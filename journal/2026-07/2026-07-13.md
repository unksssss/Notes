# 2026-07-13 星期一

## Records

### 配电房项目 — HookRope 编译错误修复链

今天在完善 HookRope.cs 的 Obi 绳索系统时，连续修复了三个编译错误：

**① `CS0200: ObiActor.solver is read only`**
```
error CS0200: Property or indexer 'ObiActor.solver' cannot be assigned to -- it is read only
```
ObiRope 的 `solver` 属性在 ObiActor 基类中是只读的，不能手动赋值。ObiRope 在 `OnEnable` 或蓝图加载时会自动与绑定的 ObiSolver 关联，不需要手动设置。

**② `Can't add component 'MeshRenderer'` 与 NullReferenceException**
```
Can't add component 'MeshRenderer' to DynamicRope because such a component is already added!
NullReferenceException: Object reference not set to an instance of an object
  HookRope.CreateRope () (at Assets/Scripts/HookRope.cs:75)
```
`ObiRopeExtrudedRenderer` 内部已经自带 `MeshRenderer` 组件，在 AddComponent 链中如果额外的 `MeshRenderer` 引起异常，会导致后续 `material` 赋值返回 null。解决方案：删除手动添加 MeshRenderer 的代码，直接通过渲染器组件访问。

**③ `CS1513: } expected`**
括号配对问题——HoldEndRoutine 或代码结构调整时遗漏了一个闭合大括号。

### 配电房项目 — HookRope 绳子耷拉地上 root cause 诊断

用户反复测试发现绳子末端不吸附在 ropeEnd 位置，总是耷拉在地面上：

```
rope particles: 17
ropeStart world: (0.17, 0.59, 2.58), ropeEnd world: (0.00, 0.28, 3.03)
```

关键发现：`ropeEnd` 的 world Y 坐标 = **0.28**（地面高度），而 `ropeStart`（Player）Y = **0.59**。Inspector 上 Rope End 字段指向了**地面上的空物体**而非 Player，导致 Pin Constraints 锚定到了地面位置。这是场景引用配置问题，非代码 bug。

**Root Cause**：HookRope 组件的 `ropeEnd` 字段在 Inspector 中拖入了地上（Y=0.28）的 GameObject，而不是 Player 身上的 Transform（Y=1.0 左右）。

### 配电房项目 — HookRope.cs 最终稳定版本

当前稳定版本（120行）采用完整的 Obi 绳索体系：

```csharp
// 建立绳索对象的骨架
var ropeObj = new GameObject("DynamicRope");
ropeObj.transform.SetParent(solver.transform);
ropeObj.transform.position = ropeEnd.position;

_rope = ropeObj.AddComponent<ObiRope>();
_rope.maxBending = maxBending;

// 渲染器 + 截面
var renderer = ropeObj.AddComponent<ObiRopeExtrudedRenderer>();
var section = ScriptableObject.CreateInstance<ObiRopeSection>();
section.CirclePreset(8);
for (int i = 0; i < section.vertices.Count; i++)
    section.vertices[i] *= ropeThickness;
renderer.section = section;

// Cursor — 动态调整绳长
_cursor = ropeObj.AddComponent<ObiRopeCursor>();
_cursor.cursorMu = 0;
_cursor.direction = true;

// 通过 Pin Constraints 固定两端
var pinConstraints = _rope.GetConstraintsByType(Oni.ConstraintType.Pin)
    as ObiConstraints<ObiPinConstraintsBatch>;
var batch = new ObiPinConstraintsBatch();
batch.AddConstraint(_rope.solverIndices[0], ec, Vector3.zero,
    Quaternion.identity, 0, 0, float.PositiveInfinity);
batch.AddConstraint(_rope.solverIndices[_rope.activeParticleCount - 1],
    sc, Vector3.zero, Quaternion.identity, 0, 0, float.PositiveInfinity);
pinConstraints.AddBatch(batch);
_rope.SetConstraintsDirty(Oni.ConstraintType.Pin);

// Update 中动态调整绳长
void Update()
{
    float dist = Vector3.Distance(ropeStart.position, ropeEnd.position);
    if (dist > _rope.restLength && dist <= maxRopeLength)
        _cursor.ChangeLength(dist);
}
```

**架构特点**：
- 绳索作为 `solver.transform` 的子物体，world/local 坐标转换通过 Obi 自动处理
- `ObiRopeCursor` 在 Update 中根据两端实际距离动态调节 `restLength`
- `Pin Constraints` 绑定粒子[0]到 ropeEnd（锚点端）、粒子[last]到 ropeStart（玩家端）
- 两端 GameObject **必须**挂载 `ObiColliderBase`（ObiCollider）组件，否则 Pin 约束无锚点
