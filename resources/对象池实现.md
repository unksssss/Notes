# 对象池实现

## 为什么需要对象池？

反复 `Instantiate` 和 `Destroy` 会：
- 产生 GC Alloc（垃圾回收压力）
- 分配和释放内存（CPU 开销大）
- 频繁触发 GC 导致卡顿

## 使用 IObjectPool<T>（Unity 2021+ 内置 API）

### 常见错误写法（踩坑）

```csharp
// ❌ 问题1：预创建的对象都是 active 的
createFunc: () => Instantiate(_prefab),

// ❌ 问题2：OnDisable + Invoke 导致循环调用
void OnDisable()
{
    Invoke(nameof(ReturnPool), 3f);
}
```

预创建时 Instantiate 出来的对象默认 active，而且 OnDisable 每次 SetActive(false) 都会触发，导致 Invoke 被反复调度，最终多次 Release 同一个对象。

### 正确写法

Pool 脚本：

```csharp
using UnityEngine;
using UnityEngine.Pool;

public class BulletPool : MonoBehaviour
{
    private IObjectPool<Bullet> _pool;
    [SerializeField] private Bullet _prefab;

    void Awake()
    {
        _pool = new ObjectPool<Bullet>(
            createFunc: () =>
            {
                var bullet = Instantiate(_prefab);
                bullet.gameObject.SetActive(false);   // 创建时就 inactive
                return bullet;
            },
            actionOnGet: bullet =>
            {
                bullet.gameObject.SetActive(true);
                bullet.StartReturnTimer(10f);          // 取出时启动倒计时
            },
            actionOnRelease: bullet =>
            {
                bullet.CancelReturnTimer();            // 回池时取消倒计时
                bullet.gameObject.SetActive(false);
            },
            actionOnDestroy: bullet => Destroy(bullet.gameObject),
            collectionCheck: true,                     // 开启重复回池检测
            defaultCapacity: 20,
            maxSize: 100
        );
    }

    public Bullet Get() => _pool.Get();
    public void Release(Bullet bullet) => _pool.Release(bullet);
}
```

Bullet 脚本：

```csharp
using UnityEngine;

public class Bullet : MonoBehaviour
{
    private BulletPool _pool;
    private float _returnTime;
    private bool _returnTimerActive;

    void Start()
    {
        _pool = FindObjectOfType<BulletPool>();
    }

    public void StartReturnTimer(float seconds)
    {
        _returnTime = seconds;
        _returnTimerActive = true;
    }

    public void CancelReturnTimer()
    {
        _returnTimerActive = false;
    }

    void LateUpdate()
    {
        if (!_returnTimerActive) return;

        _returnTime -= Time.deltaTime;
        if (_returnTime <= 0f)
        {
            _returnTimerActive = false;
            _pool.Release(this);
        }
    }
}
```

## 完整生命周期

```
游戏启动
  → 预创建 20 个对象，全部 SetActive(false)
  → Get()
    → SetActive(true) + 启动倒计时
    → 对象正常工作
  → 倒计时结束 → Release()
    → CancelTimer + SetActive(false)
    → 回到池中，等待下次 Get()
```

## 核心参数说明

| 参数 | 作用 |
|------|------|
| defaultCapacity | 预分配数量（初始 inactive） |
| maxSize | 最大上限，超出不再新建 |
| collectionCheck | 是否检查重复回池（开发阶段建议开启） |

## ⚠️ 常见坑点

1. **不要在 OnDisable 里回池** — OnDisable 在 SetActive(false) 时触发，导致循环
2. **预创建的对象要先设 inactive** — 否则游戏一开始就有一堆 active 对象在场景里
3. **回池前取消计时器** — 避免倒计时回调时对象已被其他逻辑释放
