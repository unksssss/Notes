### 配电房项目 — PlayerController 纯玩家控制模块重构

**目标文件**: `Assets/Scripts/Player/PlayerController.cs`

**问题背景**：旧的 `FirstPersonController.cs` 混入了消防模块引用（灭火烟雾粒子等），职责不清晰，且缺少下蹲功能。

**架构方案**：新建纯玩家操控核心 `PlayerController`，职责严格限定在 **移动 · 视角 · 下蹲 · 状态事件**，零消防逻辑引用。

**实现要点**：
- `[RequireComponent(typeof(CharacterController))]` 自动依赖
- 外部模块通过 C# 事件解耦：`OnJump` / `OnCrouch` / `OnRun`
- 公开状态属性：`IsGrounded` / `IsCrouching` / `IsRunning` / `IsMoving` / `Velocity` / `DesiredMove`
- **下蹲系统**：LeftControl 长按触发，`Mathf.Lerp` 平滑过渡胶囊体高度 + 相机位置跟随
- **视角**：`_xRotation`（俯仰）clamp ±80°，`_yRotation`（偏航）累加无限制
- **Alt 临时解锁光标**：编辑器友好，按住 Alt 时 `CursorLockMode.Confined`
- 蹲下时禁用跳跃
- 旧 `FirstPersonController.cs` 保留供旧场景兼容

**代码片段** — 状态事件设计：
```cs
// 消防模块通过监听事件解耦，无需修改 PlayerController
public event Action OnJump;
public event Action<bool> OnCrouch;     // true=蹲下, false=站起
public event Action<bool> OnRun;        // true=开始跑, false=停止跑
```

**踩坑**：`CharacterController.center` 的 Y 必须随高度调整，否则站起时胶囊体中心偏下，相机位置异常。

### 配电房项目 — LaserPointer 动态水柱 Tube 网格（多次迭代最终方案）

**目标文件**: `Assets/Scripts/Player/LaserPointer.cs`

**需求**：按住鼠标左键时，从枪口发射射线，起点到击中点之间生成动态水柱 Tube，末端生成 Spray 特效，松开后水柱渐消。

**最终技术方案**：每帧重建自定义 Tube 网格（无端面，无 UV 撕裂）

**核心机制**：

1. **拖尾曲线** — `curvePositions[]` 数组 + 速度衰减插值：
```cs
// 连续速度衰减：t=0 几乎瞬时、t=1 最慢
float speed = trailSpeed / (1f + t * trailDrop);
_curvePositions[i] = Vector3.Lerp(_curvePositions[i], ideal, Mathf.Clamp01(speed * dt));
```
- `trailSpeed=40`（近端响应快）
- `trailDrop=20`（远端滞后，形成自然拖尾感）

2. **Tube 网格无端面** — 环形顶点沿曲线分段，三角面只建管壁：
```cs
int vertCount = curvePoints * tubeSegments;       // 30×12 = 360 顶点
int triCount  = (curvePoints - 1) * tubeSegments * 6;  // 2088 索引
```
- `BuildCurveTriangles()` 预构建一次三角索引，每帧只更新顶点+UV

3. **UV 不拉伸** — UV.V 按累积弧长占比映射，0→1 全程覆盖，兼容 ShaderGraph 消融特效：
```cs
float v = 1f - (curveLen > 0.001f ? acc / curveLen : (float)p / (curvePoints - 1));
```

4. **环扭转消除** — `Quaternion.FromToRotation(prevTangent, fwd)` 传递上一环 up：
```cs
up = Quaternion.FromToRotation(tangents[p - 1], fwd) * up0;
up0 = up;
```

5. **溶解动画** — ShaderGraph 属性 `Vector1_3b0bddb6200046a9b085f11bb209c326`：
- 按下：Disolve 1→0（水柱渐显，`dissolveDuration=0.3s`）
- 松开：Disolve 0→1（水柱渐消），完成后 `SetActive(false)`
- Spray 粒子在水柱完全显现后才播放

6. **NormalTiling 缩放** — 水柱拉伸时法线贴图等比放大：
```cs
_cylinderMat.SetFloat(ShaderProp_NormalTiling, _baseNormalTiling * straightLen / 2f);
```

7. **UV 流动** — ShaderGraph 属性 `Vector1_ab5bbc78aace46cca47c976624c57654` 控制流动速度

**踩坑**：
- 每帧必须调用 `RecalculateNormals()` + `RecalculateBounds()`，否则视锥体裁剪会导致水柱消失
- 三角绕序必须为 `(b0, t0, b1)` / `(b1, t0, t1)` 保证正面朝外（CW winding）
- `transform` 保持 `position=(0,0,0)` / `rotation=identity` / `scale=(1,1,1)`，所有顶点用世界坐标（避免 Transform 与顶点变换叠加）

### 配电房项目 — RopeTest 场景参数调优

**场景文件**: `Assets/Scenes/RopeTest.unity`

今日在 RopeTest 场景中做了以下参数调整（已保存未提交）：

| 参数 | 旧值 | 新值 | 说明 |
|------|:----:|:----:|:----|
| Solver **damping** | 0.98 | **0** | 撤除绳索物理阻尼，模拟更活波的自然鞭打/摆动效果 |
| HookRope **maxRopeLength** | 10000 | **50** | 从"几乎无限"降为合理绳长（约半栋配电房跨度），防止多余绳索到处飘 |
| 演示方块 **scale** | (3,3,3) | **(30,30,30)** | 放大锚点参考物体，便于调试时定位 |
| 新增 **MeshFilter + MeshRenderer** | — | 6 材质 | 场景中新增了一个带多材质的参考物体 |

**重点**：`damping=0` 意味着 Obi 绳索 solver 不再对粒子速度做衰减，绳索末端的摆动会更自由、更接近真实消防水带的 whip 效果。如果摆动过大导致穿模，后续再逐步回调。
