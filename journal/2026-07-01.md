### 配电房项目 — BreakingWall.cs 破拆墙逻辑实现与修复

**目标文件**: `Assets/Scripts/Author/Wt/BreakingWall.cs`

实现了墙体破拆的核心逻辑，包含方向控制、距离衰减、HighlightPlus 集成等功能：

**① 方向控制修复**
```diff
- private Vector3 breakingDirection;
+ [SerializeField] private Vector3 _breakingDirection = new Vector3(1, 0, 0);
```
```diff
- rb.AddForce(breakingDirection * 20f, ForceMode.Impulse);
+ rb.AddForce(_breakingDirection * explosionForce * forceFactor, ForceMode.Impulse);
```
- 将 `breakingDirection` 改为 `[SerializeField]`，Inspector 可直接配置
- 去掉硬编码 `*20f`，改为 `explosionForce` 字段统一控制力大小
- 默认方向修正为 `(1,0,0)` X 正方向

**② 基于距离的受力衰减**
```cs
private void BreakFromCenter()
{
    foreach (var rb in wallFragments.GetComponentsInChildren<Rigidbody>())
    {
        rb.isKinematic = false;
        float distance = Vector3.Distance(rb.position, breakingCenter.position);
        // 距离越近力越大，超出 1m 后无力
        float forceFactor = Mathf.Clamp01(1f - distance / 1f);
        rb.AddForce(_breakingDirection * explosionForce * forceFactor, ForceMode.Impulse);
    }
}
```
- 每个碎片根据与 `breakingCenter` 的距离计算受力系数
- 距离 ≤ 0m → 全威力，距离 ≥ 1m → 无力（`Mathf.Clamp01` 保范围）

### 踩坑记录 — AddExplosionForce 与自定义方向

**问题**：之前用 `AddExplosionForce(explosionForce, breakingCenter, explosionRadius)` 时，碎片运动不受 `breakingCenter` 位置影响——距离再远都有反应。

**原因**：`AddExplosionForce` 的原理是从爆炸中心向刚体位置方向施加力，且作用于所有在半径内的刚体。如果 `breakingCenter` 离墙体很近（在半径内），则所有碎片都会受力。只看 `breakingCenter` 的位置而没有做距离范围限制是无法控制哪些碎片受力、哪些不受的。

**改用方案**：`AddForce` + 手动 `Vector3.Distance` 计算 + `Mathf.Clamp01` 做距离衰减系数。
