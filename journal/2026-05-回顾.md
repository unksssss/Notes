## 配电柜房 VR 消防演练 — 架构设计

### 项目概况

基于 **Unity 2022.3.61f1c1** 的消防 VR 培训系统，配电柜房火灾场景。使用 **OAVA Flame** 插件实现火焰系统，自建 StepManager/StepAsset 流程控制框架。

## 流程控制系统 — StepManager + StepAsset + StepBranch

### 设计思路

自定义了一套轻量级事件驱动流程控制，替代 Unity Visual Scripting 或 Dialogue System 等重型方案：

- **StepAsset** (ScriptableObject)：每个步骤的数据容器，含 stepId、任务描述、分支数组
- **StepBranch**：分支定义，含 isCorrect（正确/错误）、targetStepId、feedback
- **StepManager** (单例)：运行时调度，维护当前步骤 + 历史路径 + 事件广播

```cs
// StepAsset — ScriptableObject 驱动
public class StepAsset : ScriptableObject
{
    public string stepId;
    public string task;
    public string description;
    public StepBranch[] branches;
}

// StepBranch — 分支跳转
public class StepBranch
{
    public string branchId;
    public bool isCorrect;
    public string targetStepId;
    public string feedBack;
}
```

### UnityEvent 广播机制

用 `UnityEvent<StepAsset>` 和 `UnityEvent<StepBranch>` 在 Inspector 中可视化的方式解耦步骤响应：

```cs
public UnityEvent<StepAsset> OnStepEnter;    // 步骤进入时广播
public UnityEvent<StepBranch> OnBranchChosen; // 分支选择时广播
```

各 UI 面板订阅 `OnStepEnter` 即可响应步骤切换，无需在 StepManager 中硬编码面板逻辑。

### 经验总结

- **ScriptableObject + UnityEvent** 非常适合 VR 培训这种有限状态、线性推进的场景
- 相比完整的 FSM 框架（如 PlayMaker），这套方案更轻量、调试更方便
- Inspector 中直接看到当前步骤和历史路径（`[ReadOnly]` + `visitHistroy`），无需写额外调试工具

## OAVA Flame — 火焰系统的坑

### OAVA_IGNIS 宏

OAVA Flame 依赖 `OAVA_IGNIS` 宏定义。必须在 **Player Settings → Scripting Define Symbols** 中添加，否则灭火相关代码不会编译。

### 粒子灭火流程

灭火器粒子需要同时配置：
- **Collision 模块**：Type=World，Mode=3D，开启 Send Collision Messages
- **触发检测**：OnParticleCollision 回调中检测 FlammableObject

```cs
void OnParticleCollision(GameObject other)
{
    var flame = other.GetComponentInParent<FlammableObject>();
    if (flame != null && flame.onFire)
        _currentFlame = flame;
}
```

### 大物体的 IncrementalExtinguish 陷阱

`IncrementalExtinguish(position, startRadius, radiusIncrement)` 对大型物体（shelf）有坑：
- 每次调用会**重新计算灭火区域中心**，导致之前的灭火进度丢失
- 解决方案：**改为直接操作 `flameParticleSize` 和 `flameVFXMultiplier`**，绕过 OAVA 的面积灭火机制

```cs
// 替代 IncrementalExtinguish 的方案
_currentFlame.flameParticleSize = Mathf.MoveTowards(
    _currentFlame.flameParticleSize, 0f, Time.deltaTime * 0.3f);
_currentFlame.flameVFXMultiplier = Mathf.MoveTowards(
    _currentFlame.flameVFXMultiplier, 0f, Time.deltaTime * 0.3f);
```

### 子物体无独立 FlammableObject 时的重新点燃问题

父物体上有 FlammableObject，子物体上没有独立脚本时：
- 灭掉一个子位置的火焰 → 相邻火焰通过 FireTrigger 重新点燃该位置
- 原因：`isReignitable ` 设为 `No` 只阻止**外部**重燃，不阻止**自身 FireTrigger 内部传播**

**解决：** 使用 `ResetObj()` 彻底销毁火焰 VFX 和 FireTrigger，避免传播。

### FlammableObject 性能考虑

每个物体都挂 FlammableObject 是否影响性能？

- **CPU 端**：每个 FlammableObject 在 Update 中有较多的碰撞/灭火计算，**不建议大量挂载**（超过 20~30 个需评估）
- **GPU 端**：VFX Graph 火焰粒子由 GPU 处理，性能瓶颈在粒子总数而非 FlammableObject 数量
- **建议**：仅在需要独立的起火/灭火/重新点燃逻辑的物体上挂载

## 自定义灭火系统 — CustomParticalExtinguish

### 设计目标

绕过 OAVA 的 ParticleExtinguish（有 NullReference 问题和面积重置 bug），自己控制灭火逻辑。

```cs
public class CustomParticalExtinguish : MonoBehaviour
{
    public float extinguishSpeed = 0.3f;       // 火焰缩小速度
    public float minSizeToExtinguish = 0.03f;  // 火焰缩小到该值以下则完全灭掉
    public float rayCheckDistance = 5f;         // 射线检测距离

    private void Update()
    {
        // 粒子未喷射 → 不处理
        if (!_particle.isPlaying) { _currentFlame = null; return; }

        // 射线检测找到燃烧中的 FlammableObject
        if (_currentFlame == null || !_currentFlame.onFire)
        {
            Ray ray = new Ray(transform.position, transform.forward);
            if (Physics.Raycast(ray, out var hit, rayCheckDistance))
            {
                var flame = hit.collider.GetComponentInParent<FlammableObject>();
                if (flame != null && flame.onFire) _currentFlame = flame;
            }
        }

        // 持续缩小火焰
        if (_currentFlame != null && _currentFlame.onFire)
        {
            _currentFlame.flameParticleSize -= Time.deltaTime * extinguishSpeed;
            _currentFlame.flameVFXMultiplier -= Time.deltaTime * extinguishSpeed;

            if (_currentFlame.flameParticleSize <= minSizeToExtinguish)
            {
                _currentFlame.ResetObj();  // 彻底灭掉
                _currentFlame = null;
            }
        }
    }
}
```

### 关键设计点

- **射线检测 + 方向限制**：基于粒子发射方向检测，而非 OAVA 的全局检测（全部物体上的粒子碰撞）
- **渐进缩小**：视觉上更自然，避免火焰突然消失
- **ResetObj 终止**：包括销毁火焰 VFX 和 FireTrigger

## 泡沫粒子特效 — FoamEffect

用代码创建灭火器泡沫粒子系统，无需预制体：

```cs
// 关键配置
main.startColor = new Color(0.9f, 0.95f, 1f, 0.7f);   // 淡蓝白色
shape.shapeType = ParticleSystemShapeType.Cone;          // 锥形喷射
shape.angle = 15f;                                        // 锥角 15 度
noise.strength = 0.3f;                                    // 噪声模拟泡沫散开
```

### 经验

- Unity 默认粒子材质可以通过 `Resources.Load<Material>("Default-Particle")` 加载
- 锥形（Cone）+ 噪声模块（Noise）是模拟灭火器喷射最自然的方式
- 无法通过脚本设置材质的 `RenderMode`，需在粒子 Renderer 组件上手动切换

## 闪电特效 — LightningBolt 工具

### 用法

- 导入外部 LightningBolt 资源后，在 Script 挂载到空物体上
- 通过 `StartLightning()` / `StopLightning()` 控制起止
- 电流跟随路径需根据场景配置起始/终止点 Transform

### 与流程控制的结合

地面的电流效果由 StepManager 步骤驱动：进入某个步骤时激活闪电 → 执行完该步骤后关闭。

## 灭火器动画 — Legacy Animation 问题

`Animation.Play()` 报错：
```
The AnimationClip 'FireExtinguisher' must be marked as Legacy.
```

### 解决

选中 `.anim` 文件 → Inspector → 勾选 **Legacy**。原因是 `Animation` 组件（旧系统）不能播放非 Legacy 动画。如果要用新系统，改用 `Animator` + AnimatorController。

## 道具选择面板 — EquipmentPanel

实现了多选 Toggle 的正确/错误判断：

```cs
private bool GetResult()
{
    // 只有同时选了「绝缘手套」「绝缘靴」「绝缘操作杆」才算正确
    int correctChoice = 0;
    foreach (var s in selectedItems)
    {
        if (s == "绝缘手套" || s == "绝缘靴" || s == "绝缘操作杆")
            correctChoice++;
        else return false;  // 选了错的直接 false
    }
    return correctChoice == 3;
}
```

## 总结 — 踩过的坑

| 坑 | 原因 | 解决 |
|---|---|---|
| `OnParticleCollision` NullReference | OAVA 的 ParticleExtinguish 没配粒子系统 | 自定义 CustomParticalExtinguish |
| `IncrementalExtinguish` 灭火进度丢失 | 每次调用重算灭火区域中心 | 改为直接操作 flameParticleSize |
| 子物体灭火后被重新点燃 | FireTrigger 内部传播不受 isReignitable 限制 | ResetObj 销毁所有 FireTrigger |
| `Animation.Play()` 报 Legacy 错误 | Animation 组件不能播放非 Legacy Clip | 勾选 Legacy 或改用 Animator |
| 泡沫粒子粉色 | 默认材质丢失 | 手动设置 renderer.material |
