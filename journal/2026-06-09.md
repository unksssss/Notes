### 配电房项目 — BezierHose：贝塞尔曲线水管实时铺设

从预创建圆柱体池改为三次贝塞尔曲线 + 圆柱体拉伸的实时管道铺设方案。

```cs
// 核心：贝塞尔采样 + 圆柱体定位
for (int i = 0; i < segmentCount; i++)
{
    float t = (float)i / segmentCount;
    float tNext = (float)(i + 1) / segmentCount;

    Vector3 pos = CubicBezier(p0, p1, p2, p3, t);
    Vector3 nextPos = CubicBezier(p0, p1, p2, p3, tNext);
    Vector3 dir = nextPos - pos;

    Transform seg = _segments[i];
    seg.position = pos + dir * 0.5f;
    seg.rotation = Quaternion.LookRotation(dir) * Quaternion.Euler(90, 0, 0);
    seg.localScale = new Vector3(pipeRadius, dir.magnitude, pipeRadius);
}
```

**关键设计：**
- 控制点自动贴地（`ToGround`），在XZ平面上根据 `controlOffset` 偏移
- **Rotation 修复**：`LookRotation(dir) * Quaternion.Euler(90, 0, 0)` ——圆柱体Z轴（长轴）沿曲线方向横滚，而不是圆面朝上
- **Y轴拉伸**：`localScale.y = dir.magnitude` 让每段圆柱体长度匹配曲线段长度
- 只在端点移动时重算（`_prevStart` / `_prevEnd` 脏检查），避免每帧重建

### 配电房项目 — MultiPointBentCylinder：程序化弯曲圆柱体网格（新文件）

放弃多段圆柱体方案，改用**整条管道一个 Mesh**，通过 Catmull-Rom 样条线插值 + 环形截面生成。

```cs
// Catmull-Rom 插值（比贝塞尔更适合"经过所有路径点"）
private Vector3 GetCatmullRomPoint(int segmentIndex, float t)
{
    Vector3 p0 = GetWorldPathPoint(segmentIndex - 1);
    Vector3 p1 = GetWorldPathPoint(segmentIndex);
    Vector3 p2 = GetWorldPathPoint(segmentIndex + 1);
    Vector3 p3 = GetWorldPathPoint(segmentIndex + 2);

    float t2 = t * t;
    float t3 = t2 * t;
    return 0.5f * (
        2f * p1 +
        (-p0 + p2) * t +
        (2f * p0 - 5f * p1 + 4f * p2 - p3) * t2 +
        (-p0 + 3f * p1 - 3f * p2 + p3) * t3);
}
```

**特点：**
- 路径点序列：起点 → 若干 bendPoint → 终点，Catmull-Rom 保证曲线经过每个路径点
- 环形截面顶点生成（`_radialSegments` 控制圆滑度）
- 世界坐标 → `transform.InverseTransformPoint` 转本地坐标生成 Mesh
- 双面渲染（`_doubleSided`）+ 可选封端盖（`_capEnds`）
- `[RequireComponent(typeof(MeshFilter), typeof(MeshRenderer))]`

### 配电房项目 — ThreePointBentCylinder：三点二次贝塞尔弯曲圆柱体（新文件）

二次贝塞尔曲线版本，bendPoint → 控制点的反向计算。

```cs
// bendPoint 转换为真正的贝塞尔控制点（t=0.5经过bendPoint）
private static Vector3 GetControlPointThroughBendPoint(Vector3 start, Vector3 bend, Vector3 end)
{
    // 二次贝塞尔 t=0.5 时：0.25*start + 0.5*control + 0.25*end = bend
    // 反推：control = 2*bend - 0.5*(start+end)
    return 2f * bend - 0.5f * (start + end);
}
```

**关键设计：**
- **bendPlaneNormal**：用三个点确定的平面法线锁定截面朝向，防止沿曲线旋转扭曲
- `OnValidate` 支持 Inspector 编辑时实时预览
- `[ContextMenu("Rebuild Mesh")]` 右键菜单手动重建
- `SetControlPoints()` 接口供 `FireTruck` 运行时调用

### 配电房项目 — FireTruck：消防车水带分段落铺设（重构）

从"提前知道路径"改为"玩家实时触发→记录路径→分段落铺设"。

```cs
// 玩家每走出 bendPointInterval 距离，固定当前段，创建新段
if (Vector3.Distance(_currentSegment.StartPoint.position, 
    _currentSegment.EndPoint.position) >= _bendPointInterval)
{
    _currentSegment = CreateSegment(_currentSegment.EndPoint.position);
    UpdateSegments();
}
```

**BendPoint 自动计算算法：**
1. 取当前段 start→end 连线中点 mid
2. 作该连线的**法线**方向（XZ平面）
3. 与下一段 start→end 连线求**二维直线交点**
4. 交点偏移限制在 `_maxBendOffset`（默认2m）内

```cs
// 限制偏移，避免水带大幅外甩
Vector2 offset = intersection - midXZ;
if (offset.magnitude > maxOffset)
    intersection = midXZ + offset.normalized * maxOffset;
```

**运行时对象管理：**
- 每个 `HoseSegment` 包含 startPoint/endPoint/bendPoint 三个 `Transform` + 一个 `ThreePointBentCylinder`
- `ClearHose()` 销毁所有段 + 运行时控制点
- 玩家离开触发器不停止绘制，水带继续跟随

### 配电房项目 — SingleCylinderHoseFromFireTruck：双Mesh优化水带（新文件）

性能优化版本——**Baked + Active 双 Mesh 策略**：

| Mesh | 重建策略 | 用途 |
|------|---------|------|
| BakedHoseMesh | 玩家每走出 `_bendPointInterval` 重建一次 | 已固定的长水带 |
| ActiveHoseMesh | 每帧重建 | 最后固定点 → 玩家当前位置 |

**优势：** 避免路径变长后每帧重建整条 Mesh。Active 段永远只有 2 个路径点（最后固定点 + 玩家），重建开销恒定。

### 配电房项目 — 踩坑：Cylinder 旋转/拉伸/首尾不相接

今天在 BezierHose 上遇到三个连环坑：

| 问题 | 根因 | 修复 |
|------|------|------|
| 圆柱体圆面朝上（竖着） | 默认 rotation 使 Z 轴朝上 | `LookRotation(dir) * Euler(90,0,0)`，让 Z 轴沿曲线方向 |
| 圆柱体长度不对 | localScale 没在长轴拉伸 | `localScale.y = dir.magnitude`，在 Y 轴拉伸 |
| 圆柱体之间首尾不相接 | 多段独立圆柱体很难精确对接 | **放弃多段方案**，改用单个 Mesh 弯曲（MultiPointBentCylinder） |

> **教训：** 多段圆柱体首尾相接在实际中极难完美对齐——每个圆柱体的起点是上一段的终点位置，但 rotation 的微小偏差和数据精度问题会在连接处产生明显的视觉缝隙或错位。单个程序化 Mesh 彻底解决了这个问题。
