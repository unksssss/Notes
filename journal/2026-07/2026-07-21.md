### 配电房项目 — EquipmentChoosePlus 装备选择界面三件套完善

**目标文件：**
- `Assets/Scripts/Author/Wt/UI/EquipChoosePlus/EquipmentChoosePlus.cs`
- `Assets/Scripts/Author/Wt/UI/EquipChoosePlus/EquipmentItemPlus.cs`
- `Assets/Scripts/Author/Wt/UI/EquipChoosePlus/EquipNeededItem.cs`

**问题描述：** 装备选择界面的核心交互逻辑实现——左侧装备列表可拖拽到右侧所需装备区，支持拖拽放置、双击增减、+/- 按钮增减三种交互方式，同时需要状态反馈（正确/错误/数量不足）。

**实现方案：**

**① EquipmentChoosePlus.cs（主控制器）**
- `InitAllEquipLst()` — 从 Resources 加载 `EquipmentItemPlus` 预制体实例化至左侧列表，绑定 Toggle 选中回调
- `InitNeededEquipLst()` — 实例化右侧必备装备项，初始状态设为 `sprite_Error`（问题图标）
- `ItemPlusOrMinus()` — 核心增减方法：匹配到所需装备时更新数量并校验，超出配额的装备标记 `sprite_Wrong`
- `ItemDragEnd()` — 拖拽放置入口：如果在所需装备列表内则调用 `ItemPlusOrMinus` 增加，否则调用 `AddWrongItem` 添加错误项
- `SetTipsText()` — 鼠标悬停提示框，定位到状态图标位置，显示"种类正确数量正确"/"种类正确数量错误"/"种类错误"

```cs
public void ItemPlusOrMinus(EquipDefinition equip, bool isAdd = true)
{
    if (equip == null) return;
    foreach (var neededEquip in equipNeededLst)
    {
        if (neededEquip.equip == equip)
        {
            equipNeededDic[equip].Num += isAdd ? 1 : -1;
            equipNeededDic[equip].SetStatus(
                equipNeededDic[equip].Num == neededEquip.num
                    ? sprite_Right    // 数量正确 ✅
                    : sprite_Error);  // 数量错误 ⚠️
            return;
        }
    }
    // 不属于所需装备 — 标记为错误项
    if (!equipNeededDic.TryGetValue(equip, out var item)) return;
    item.Num += isAdd ? 1 : -1;
    if (item.Num == 0) { /* 移除项 */ }
    else { item.SetStatus(sprite_Wrong); }
}
```

**② EquipmentItemPlus.cs（可拖拽物品项）**
- 实现 `IBeginDragHandler / IDragHandler / IEndDragHandler` 三接口
- 拖拽时克隆一个 `img_DraggingIcon` 跟随鼠标，用 `RectTransformUtility.ScreenPointToLocalPointInRectangle` 转换坐标
- 拖拽结束用 `EndAreaCheck()` 检测是否落入右侧区域（`RectTransform.Contatins` 自定义扩展方法）

```cs
public void OnEndDrag(PointerEventData eventData)
{
    // 命中目标区域 → 回调通知 EquipmentChoosePlus
    if (EndAreaCheck())
    {
        EquipmentChoosePlus.Instance.ItemDragEnd(equip);
        // 还原拖拽图标
    }
    else
    {
        // 未命中 → DoTween 动画回到原位
        img_DraggingIcon.transform.DOMove(img_icon.transform.position, 0.3f)
            .onComplete += () => { /* 还原状态 */ };
    }
}
```

### 配电房项目 — RopeTest 场景参数调优

**目标文件：** `Assets/Scenes/RopeTest.unity`

**参数调整记录：**

| 参数 | 旧值 | 新值 | 说明 |
|------|------|------|------|
| `maxRopeLength` | 10000 | 50 | 绳索最大长度合理化，避免超出场景范围 |
| `damping` | 0.98 | 0 | 取消阻尼，使绳索物理响应更灵敏 |
| AnchorCube Scale | (3,3,3) | (30,30,30) | 放大锚点碰撞体，提升绳索交互检测稳定性 |
| 新增物件 | — | SM_楼梯间B | 场景扩充，位置 (-23.53, 0, 5.37) |

**根因分析：** 绳索模拟出现不稳定抖动——`damping=0.98` 导致运动衰减过度，物理响应迟缓；锚点碰撞体过小导致粒子检测部分漏检。
