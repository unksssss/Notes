### UGUI — Toggle 选中/不选中切换双图（零代码方案）

**问题**：Toggle 选中时显示图 A，不选中时显示图 B，不想写代码。

**方案**：利用 Toggle.Graphic 的显隐机制（两层面板叠加）

**层级结构**：
```
Toggle
├── BgOff (Image)          ← 未选中图，始终显示
│   └── BgOn (Image)       ← 选中图，拖到 Toggle.Graphic 字段
└── Label
```

**配置**：
1. BgOn 的 Source Image = 选中图，完全覆盖 BgOff
2. Toggle 的 Graphic 字段 = BgOn 的 Image 组件
3. Toggle Transition 保持 Fade（默认）

**原理**：`Toggle.InternalToggle()` 内部调用 `graphic.canvasRenderer.SetAlpha(isOn ? 1f : 0f)`，选中时 BgOn 透明度 1（显示），未选中时透明度 0（隐藏），露出底层的 BgOff。

**备选方案**：代码 `onValueChanged` 切换 `Image.sprite`，适合两图大小不一致的场景。

### UGUI — Transform.SetParent "data corruption" 报错

**报错**：
```
Transform resides in a Prefab asset and cannot be set to prevent data corruption
```

**根因**：`Resources.Load<T>()` 返回的是**磁盘上的 Prefab 资源引用**，不是场景实例。直接在 Prefab 资源上调用 `SetParent` 会触发 Unity 的保护机制。

**修复**：
```cs
// ❌ 错误
var item = Resources.Load<EquipmentItem>("EquipItem");
item.transform.SetParent(content.transform);

// ✅ 正确：先实例化
var prefab = Resources.Load<EquipmentItem>("EquipItem");
var item = Instantiate(prefab, content.transform);
```

**知识点**：`Resources.Load` + `Instantiate` 是标准组合，前者加载资源，后者创建实例。

### UGUI — ScrollView 水平滚动设置

**水平拖动设置**：Scroll Rect 组件 → 勾选 **Horizontal**，取消勾选 **Vertical**。

**删 Scrollbar 的影响**：不影响拖动功能。Scrollbar 只是"可视化指示器"，拖动逻辑在 Scroll Rect 组件中。删后只需清空 Scroll Rect 上 Horizontal/Vertical Scrollbar 的引用字段，避免 NullReference 警告。

### UGUI — Horizontal Layout Group "spacing=0 但间隔很大" 之谜

**根因**：**Child Force Expand** 默认勾选，会把所有剩余空间均匀分摊到每个子物体上。内容越少，每个子物体被撑得越大，空隙看起来越大。

**修复**：
1. ❌ 取消勾选 **Child Force Expand**
2. ✅ **Child Alignment** = Upper Left（左对齐）
3. Spacing 填固定值
4. Content 上挂 **Content Size Fitter** → **Horizontal Fit** = Preferred Size

**原理**：Child Force Expand 本质是对每个子物体的 `FlexibleWidth` 赋值为 1，使它们瓜分父级的剩余空间。

### UGUI — ScrollView 编程接口

```cs
// 归一化位置 (0~1)
scrollRect.normalizedPosition = new Vector2(0.5f, 1f);   // 水平居中，垂直置顶
scrollRect.horizontalNormalizedPosition = 0f;             // 最左
scrollRect.horizontalNormalizedPosition = 1f;             // 最右

// 像素距离 → normalized 换算
float scrollable = content.rect.width - viewport.rect.width;
float delta = pixelDistance / scrollable;

// 平滑滚动（Mathf.SmoothDamp）
_targetH = Mathf.Clamp01(_targetH + delta);
scrollRect.horizontalNormalizedPosition = Mathf.SmoothDamp(
    current, _targetH, ref _velocity, smoothTime);
```

**ScrollStepController**：点击按钮步进滚动，支持像素距离 + 平滑动画。

### UGUI — EventTrigger 全解

**EventTrigger vs UGUI 接口**：

| 选型原则 | 场景 |
|:--------|:----|
| `IPointerClickHandler` 接口 | 自己的脚本处理自己的点击 |
| EventTrigger | 给别人的控件加事件、运行时动态绑定、5+ 种事件时避免类太长 |
| `IBeginDragHandler` + `IDragHandler` + `IEndDragHandler` | 拖拽面板 |

**EventTrigger 不生效排查清单**：
1. **Image.Raycast Target** 没勾 ← 最常见
2. Canvas 上缺 **GraphicRaycaster**
3. EventTrigger 组件**没加 Event Type Entry**（只有壳没事件）
4. 被上层 UI 遮挡（后渲染的子物体截断射线）
5. 父物体是 Button，Button 吃掉了 PointerClick 事件（换 PointerDown 或把 EventTrigger 放 Button 上）

**代码添加 EventTrigger**：
```cs
var entry = new EventTrigger.Entry();
entry.eventID = EventTriggerType.PointerClick;
entry.callback.AddListener((data) => { /* 回调 */ });
trigger.triggers.Add(entry);
```

### UGUI — 锚点设置（底部条）

**底部工具条**：锚点预设 = **Bottom – Stretch**
- Min: (0, 0), Max: (1, 0)
- Pivot: (0.5, 0)
- Pos Y = 边距, Height = 条高度
- Left / Right = 0（左右顶边）

**锚点预设面板**：
- **按住 Alt**：锚点 + 子物体位置/大小同步偏移（调位置用）
- **关注**：锚点 + Pivot 同步移动（调旋转轴用）
- **都不勾**：只改锚点（子物体不动）

### UGUI — 屏幕自适应

**Canvas Scaler 配置**：
```
UI Scale Mode → Scale With Screen Size
├── Reference Resolution → 1920 × 1080
├── Screen Match Mode → Match Width or Height
│   └── Match → 0.5（0=按宽，1=按高）
└── Reference Pixels Per Unit → 100
```

**锚点组合建议**：底部工具条用 Bottom Stretch，地图用 Stretch Stretch（四边贴边），装备 Item 用 Center Center + 固定大小。

### 装备系统 — EquipFinish() 代码优化

**原代码**：两层 foreach 嵌套两个 Where LINQ，可读性差，`break` 多余。

**优化方案**：
```cs
// 方案 A：LINQ + FirstOrDefault
return equipItems
    .Where(item => item.toggle.isOn)
    .Select(item => equips.FirstOrDefault(e => e.name == item.txt_itemName.text))
    .Where(e => e != null)
    .ToList();

// 方案 B：foreach + Find（更好读）
var result = new List<Equip>();
foreach (var item in equipItems)
{
    if (!item.toggle.isOn) continue;
    var match = equips.Find(e => e.name == item.txt_itemName.text);
    if (match != null) result.Add(match);
}
return result;
```

`List<T>.Find()` 找到第一个即停，等价于内层循环 + break。

### 装备系统 — 枚举替代方案（数据源统一设计）

**问题**：`EquipmentType` 枚举在装备种类增多时不灵活——新增要改代码、重新编译、不可动态扩展。

**方案对比**：

| 方案 | 优点 | 缺点 |
|:---|:----|:----|
| 枚举 | 编译期安全检查 | 新增改代码，不可扩展 |
| string ID | 灵活，零代码改配 | 无编译期检查，拼错不报错 |
| **ScriptableObject** ⭐ | 新增=右键新建.asset，可扩展字段，支持配置化 | 中等工作量 |

**选择**：ScriptableObject 方案。

核心数据类：
```cs
[CreateAssetMenu(menuName = "消防仿真/装备定义")]
public class EquipDefinition : ScriptableObject
{
    public string id;
    public string displayName;
    public Sprite icon;
    public GameObject modelPrefab;
    public Vector3 modelLocalPosition;
    public Vector3 modelLocalEuler;
}
```

icon 和 modelPrefab 指向不同文件（.png vs .prefab/.fbx），不存在同步风险。  
modelPrefab 可以指向 .fbx，Unity 自动将其视为预制体引用。  
UI 层只读 icon/displayName，不引用 modelPrefab 字段——不读就不消耗。

### 装备系统 — 装备模型位置方案

**问题**：`Instantiate(equip.modelPrefab, this.transform)` 后模型 transform 被重置为 (0,0,0)，因为 prefab 文件根 transform 默认全零。

**最终方案**：**把工具模型直接放在 Equipment 预制体下**，在预制体里调好位置。代码只遍历子物体，不复用 `modelPrefab`：

```cs
public void Init(List<EquipDefinition> equips)
{
    modelDic = new Dictionary<string, GameObject>();
    foreach (Transform child in transform)
    {
        child.gameObject.SetActive(false);
        modelDic[child.name] = child.gameObject;
    }
    foreach (var equip in equips)
    {
        if (modelDic.TryGetValue(equip.id, out var model))
            model.SetActive(true);
    }
}

public void Equip(string id)
{
    // 关全部 → 开指定
    foreach (var kvp in modelDic) kvp.Value.SetActive(false);
    if (modelDic.TryGetValue(id, out var target))
        target.SetActive(true);
}
```

**out var target 生命周期**：`target` 是局部变量，出了 if 块就不可见了。但 `modelDic` 里的引用仍然指向 GameObject 对象，下次调用 `TryGetValue` 仍可取出。

### C# 常见空引用 — Dictionary 没 new

**报错**：
```
NullReferenceException: Object reference not set to an instance of an object
  at EquipmentMgr.Init (...) modelDic.Add(equip.id, item);
```

**根因**：
```cs
private Dictionary<string, GameObject> modelDic;  // ← null！
```

声明后没有 `= new Dictionary<...>()`，调用 `.Add()` 时直接炸。

**修复**：
```cs
private Dictionary<string, GameObject> modelDic = new Dictionary<string, GameObject>();
```

**经验**：所有引用类型字段（Dictionary、List、array）只要声明后立即可用，一定要初始化。或在 `Awake()` / `Start()` / `Init()` 中确保 new。
