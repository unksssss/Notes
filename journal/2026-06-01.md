## Test01 — 2D 图片轮播模拟 3D 旋转（RotationDiagram2D）

### 核心思路

用纯数学计算在 2D Canvas 上模拟一圈图片绕 Y 轴旋转的效果——不需要 3D 模型、不需要透视相机。

三个维度模拟深度感：
- **X 位置**：用分段线性函数逼近圆形投影到直线
- **Scale**：中间最大（靠近观察者）、两边最小（远离观察者）
- **SiblingIndex**：中间图片盖住两边图片，模拟前后遮挡

### 位置计算 — 分段线性逼近圆弧

```cs
private float GetX(float ratio, float length)
{
    // ratio: 0~1，表示图片在圆周上的位置（0=最左，0.5=中间，1=最右）
    if (ratio <= 0.25f)
        return ratio * length;           // 左侧远离中心 → 左移
    else if (ratio <= 0.75f)
        return (0.5f - ratio) * length;  // 中间半圈 → 从右向左扫
    else
        return (ratio - 1f) * length;    // 右侧远离中心 → 右移
}
```

**原理：** 周期为 1 的锯齿波函数。当图片在 0→1 间循环时，X 坐标形成一个连续的封闭环路——图片从左端消失的同时从右端出现，这就是"环形"的数学本质。

### 缩放计算 — 越靠近中间越大

```cs
private float GetScaleTimes(float ratio, float max, float min)
{
    float deltaScale = (max - min) / 0.5f;
    if (ratio < 0.5f)
        return max - deltaScale * ratio;    // 从最大逐渐减小
    else
        return min + (ratio - 0.5f) * deltaScale;  // 从最小逐渐增大
}
```

一个 V 形函数：ratio=0.5（中间）时 scale=max，ratio=0/1（两端）时 scale=min。

### 拖动交互 — C# 事件驱动

```cs
// RotationDiagramItem 实现 IDragHandler, IEndDragHandler
public void OnDrag(PointerEventData eventData)
{
    delta += eventData.delta.x;  // 累加拖拽距离
}

public void OnEndDrag(PointerEventData eventData)
{
    OnMove?.Invoke(delta);  // 通过 event Action<float> 通知 RotationDiagram2D
    delta = 0;
}
```

RotationDiagram2D 收到 delta 后根据正负决定旋转方向：

```cs
private void Change(int symbol)  // symbol: 1=右拖(图片右移), -1=左拖
{
    for (int i = 0; i < items.Count; i++)
    {
        items[i].ChangePosId(symbol, items.Count);  // 更新逻辑位置
        items[i].SetData(itemPosDatas[items[i].PosId]); // 刷新视觉位置+缩放
    }
}
```

### PosId 循环 — 模运算实现首尾相接

```cs
public void ChangePosId(int symbol, int totalItemNum)
{
    PosId += symbol;
    if (PosId >= totalItemNum)
        PosId %= totalItemNum;      // 超出末尾 → 回到开头
    else if (PosId < 0)
        PosId += totalItemNum;      // 超出开头 → 跳到末尾
}
```

### DOTween 平滑动画

```cs
public void SetData(ItemPosData itemPosData)
{
    Rect.DOAnchorPos(Vector2.right * itemPosData.X, _aniTime);
    Rect.DOScale(Vector3.one * itemPosData.ScaleTimes, _aniTime);
    // 动画播到一半时调整渲染顺序，避免遮挡突变
    StartCoroutine(Wait(itemPosData));
}

IEnumerator Wait(ItemPosData itemPosData)
{
    yield return new WaitForSeconds(_aniTime / 2);
    Rect.SetSiblingIndex(itemPosData.Order);  // 中间图片压住两边
}
```

### 架构总结

```
RotationDiagram2D（控制器）
  ├── 计算所有位置的 X/Scale/Order → itemPosDatas
  ├── 监听 RotationDiagramItem.OnMove 事件
  └── 拖拽时 → 循环更新所有 item 的 PosId + 刷新位置

RotationDiagramItem（单个图片）
  ├── 实现 IDragHandler / IEndDragHandler → 上报拖拽量
  ├── 暴露 event Action<float> OnMove 给控制器
  └── DOTween 驱动 RectTransform 的位置和缩放动画
```

### 技术点

- **纯数学替代 3D**：用分段函数 + V 形缩放逼近圆形透视，省掉 3D 渲染开销
- **事件解耦**：Item 只管"我被拖了多远"，Controller 管"所有图片怎么动"
- **模运算闭环**：`PosId %= totalItemNum` 实现无限循环
- **SiblingIndex 时机**：动画播一半才切层级，避免在 X 位置还没移动到位时就换遮挡顺序导致视觉跳变
- **DOTween 驱动**：`DOAnchorPos` + `DOScale` 替代 Update 手写插值

## Test01 — CalcuData 预计算 + LINQ 排序管线

### 为什么预计算而不是实时算

`CalcuData()` 在 `Start()` 中一次性算出所有位置的所有数据（X、Scale、Order），存到 `itemPosDatas` 列表里。拖拽旋转时只做索引映射——`items[i].SetData(itemPosDatas[items[i].PosId])`——不需要重新计算。

```cs
private void CalcuData()
{
    float length = (itemSize.x + offSet) * items.Count;
    float ratioOffSet = 1 / (float)items.Count;  // 每张图的 ratio 步长
    List<ItemData> itemDataList = new List<ItemData>();
    float ratio = 0;

    for (int i = 0; i < items.Count; i++)
    {
        // 对每个 ratio 预计算 X 和 Scale
        ItemPosData itemPosData = new ItemPosData();
        itemPosData.X = GetX(ratio, length);
        itemPosData.ScaleTimes = GetScaleTimes(ratio, maxScale, minScale);
        itemPosDatas.Add(itemPosData);

        // 记录 item 和位置数据的映射
        ItemData itemData = new ItemData { PosID = i };
        itemDataList.Add(itemData);
        items[i].PosId = i;
        ratio += ratioOffSet;
    }

    // 🔑 关键：按 ScaleTimes 排序 → 小的在前（离观察者远），大的在后（离观察者近）
    itemDataList = itemDataList.OrderBy(x => itemPosDatas[x.PosID].ScaleTimes).ToList();

    for (int i = 0; i < itemDataList.Count; i++)
        itemPosDatas[itemDataList[i].PosID].Order = i;  // Order 就是 SiblingIndex
}
```

**设计要点：**
- **数据与表现分离**：`ItemPosData` 存计算结果，`SetData` 只管把数据应用到 Transform
- **LINQ `.OrderBy` 决定遮挡顺序**：Scale 最小的图片 Order=0 渲染在最底层，Scale 最大的 Order 最大渲染在最顶层——自动实现"近大远小 + 近遮远"的 3D 深度感
- **O(1) 拖拽响应**：拖拽时只做 `items[i].SetData(itemPosDatas[items[i].PosId])`，纯索引映射，零计算

### 模板实例化模式

```cs
private void CreateItem()
{
    GameObject tempitem = CreatTemplate();  // 创建带 RectTransform+Image+RotationDiagramItem 的模板
    foreach (var sprite in sprites)
    {
        var item = Instantiate(tempitem).GetComponent<RotationDiagramItem>();
        item.SetParent(transform);
        item.SetSprite(sprite);
        item.OnMove += Change;  // 订阅拖拽事件
        items.Add(item);
    }
    Destroy(tempitem);  // 用完即毁
}
```

**为什么这样做：** 相比在循环里 `new GameObject` + `AddComponent` 三次，先造一个模板再 Instantiate 更干净——所有组件的初始状态（RectTransform size、Image 材质等）一次性配好，不会漏加组件。

### ⚠️ 注意事项
