### 配电房项目 — HookRope 改为一次性全长生成（替代动态延长）

**目标文件**: `Assets/Scripts/HookRope.cs`

**问题背景**：之前使用 ObiRopeCursor 动态延长绳索——玩家走远时 `_cursor.ChangeLength(dist)` 添加新粒子。但 cursor 新粒子永远沿直线添加，来不及受重力下垂，导致玩家端绳索始终绷直，物理表现不真实。

**根因分析**：cursor 在运行时添加的粒子从 rope 末端（锚点端）沿直线延伸，物理 solver 需要若干帧才能让新粒子受重力下垂，在玩家持续移动的场景下永远来不及。

**修复方案**：改为一次性生成全长绳索。blueprint path 直接生成 `maxRopeLength = 20m` 长度，多余部分从高处落下自然堆积在 AnchorCube 后。

关键改动（commit `f7d042f`，比 `600c994` 的 cursor 版精简 33 行）：

```cs
// 移除 ObiRopeCursor 字段和相关逻辑
// private ObiRopeCursor _cursor;  ← 删除

// Path 控制点沿锚点方向 + 向上偏移，让多余绳索自然下垂堆积
Vector3 toAnchor = ropeStart.position - ropeEnd.position;
Vector3 dir = new Vector3(toAnchor.x, Mathf.Abs(toAnchor.y) + 1f, toAnchor.z).normalized;

_blueprint.path.Clear();
_blueprint.path.AddControlPoint(Vector3.zero, -dir, dir, Vector3.up, particleMass, 0.1f, 1, filter, Color.white, "start");
_blueprint.path.AddControlPoint(dir * maxRopeLength, -dir, dir, Vector3.up, particleMass, 0.1f, 1, filter, Color.white, "end");
_blueprint.path.FlushEvents();
_blueprint.GenerateImmediate();
_rope.ropeBlueprint = _blueprint;

// Pin 约束旋转柔度 = ∞，防止喷枪旋转传递给绳索
var batch = new ObiPinConstraintsBatch();
batch.AddConstraint(_rope.solverIndices[0], ec, Vector3.zero, Quaternion.identity,
    0, float.PositiveInfinity, float.PositiveInfinity);   // 旋转刚度=∞
batch.AddConstraint(_rope.solverIndices[_rope.activeParticleCount - 1], sc, Vector3.zero, Quaternion.identity,
    0, float.PositiveInfinity, float.PositiveInfinity);
```

**踩坑记录**：
- Pin 约束 `rotationStiffness` 必须设为 `float.PositiveInfinity`，否则喷枪旋转会扭曲绳索粒子
- 绳索 transform.position 设为 ropeEnd（玩家端）位置，使 CP0 = (0,0,0) 对齐玩家端世界坐标
- `GenerateImmediate()` 替代 `yield return _blueprint.Generate()`，避免 CoroutineJob.ProgressInfo yield 卡死

### 配电房项目 — Obi 重力与端点固定规则确认

**经验总结**（记录到 MEMORY.md）：
- 重力必须设 `solver.gravity`（顶层字段），`solver.parameters.gravity` 每帧被 ObiSolver.BeginStep 覆盖
- 端点固定可用两种方式：
  - **ObiParticleAttachment（Static）**：直接设 invMass=0 + 每帧更新位置，不依赖 solver 约束系统
  - **Pin 约束（当前方案）**：更接近原生 Obi 效果，允许弹性互动；旋转刚度设 ∞ 防止扭矩传递
- `blueprint` 赋值后 `solverIndices` 立即可用，无需等待 `isLoaded`

### 配电房项目 — 第一人称控制器搭建

**目标文件**: `Assets/Scripts/Player/FirstPersonController.cs`

新增测试用 FPS 控制器，挂载 CharacterController 组件：

```cs
// 移动
Vector3 move = transform.right * horizontal + transform.forward * vertical;
_controller.Move(move * speed * Time.deltaTime);

// 跳跃
_velocity.y = Mathf.Sqrt(jumpForce * -2f * gravity);

// Alt 键临时释放鼠标（查看场景）
bool altHeld = Input.GetKey(KeyCode.LeftAlt) || Input.GetKey(KeyCode.RightAlt);
if (altHeld) {
    Cursor.lockState = CursorLockMode.Confined;
    Cursor.visible = true;
    return;
}
```

**功能列表**：
- WASD 移动 + Shift 跑步 + 空格跳跃（CharacterController）
- 鼠标视角（灵敏度可调，俯仰 ±80°）
- Alt 键临时释放光标（UI 交互用）
- 左键喷射灭火烟雾粒子（`fireSmoke.Play()` / `.Stop()`）

### 配电房项目 — OAVA-Flame (Ignis Interactive Fire System) 研究

**目录**: `Assets/OAVA-Flame/` | **命名空间**: `Ignis`

核心组件梳理：

| 组件 | 职责 |
|------|------|
| **FlameEngine** (Singleton) | 全局控制器，管理 9 种 VFX 变体、LOD、暂停/恢复 |
| **FlammableObject** | 每个可燃物挂载，控制点火/灭火/材质动画/事件 |
| **FireTrigger** | OverlapBox 检测火焰传播 |
| **FlameEventInvoker** | UnityEvent：Ignited / Extinguished / BurntOut / etc. |

9 种 VFX 变体：Wild / Gentle / LegacyOriginal / LegacyLightweight / Lively / Simple / OldSchool / OnlySmoke / Custom

**项目已有自定义**：
- `CustomParticalExtinguish.cs` — 灭火器粒子系统，`ExtinguishFully()` 完全灭火
- `FlammableObject.charBlendMultiplier` — 自定义焦炭化速度乘数
- `FlammableObject.ExtinguishFully()` — 清理 VFX + 标记熄灭，不重置 extinguished 状态
