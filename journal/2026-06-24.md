### 配电房项目 — GameRoot 游戏入口初始化架构

**目标文件**: `Assets/Scripts/GameRoot.cs`

搭建了项目的中央初始化节点，采用经典 Singleton + `[SerializeField]` 注入模式：

```cs
public class GameRoot : MonoBehaviour
{
    public static GameRoot Instance { get; private set; }
    [SerializeField] public FirstPersonController _player;
    [SerializeField] private EquipmentPanel _equipmentPanel;
    [SerializeField] private ScoutPanel _scoutPanel;
    [SerializeField] private EnvironmentMgr _environmentMgr;

    private void Init()
    {
        // _player.Init();       // 暂时禁用
        _equipmentPanel.Init();
        _scoutPanel.Init();
        _environmentMgr.Init();
        _stepManager.Init();
        _stepManager.StartFlow();  // 启动步骤流程
    }
}
```

**架构要点**：
- GameRoot 不继承任何业务逻辑，只负责**时序编排**——按依赖顺序初始化各子系统
- `StepManager.StartFlow()` 在所有子系统就绪后才调用，避免步骤触发时依赖未就绪
- `_player.Init()` 被注释掉，推测 Player 初始化还在 FirstPersonController 内部自管理
- `SetLightingActive()` 被注释掉，环境光控制在 EnvironmentMgr 中按步骤响应

### 配电房项目 — Trigger 步骤触发 + 提示系统

**目标文件**: `Assets/Scripts/Trigger.cs`

实现了基于触发器 + 步骤索引的交互提示系统。玩家走进触发区时，根据当前 step 显示不同的操作提示：

```cs
public class Trigger : MonoBehaviour
{
    public UnityEvent callBack;
    public TipsPanel tipsPanel;
    public int step = 0;

    private void Start()
    {
        callBack.AddListener(() =>
        {
            switch (step)
            {
                case 0:
                    tipsPanel.SetTipsText("the shelf is electric!!! please cut off the power supply first!!!");
                    break;
                case 1:
                    tipsPanel.SetTipsText("cut off the power supply");
                    break;
            }
        });
    }

    private void OnTriggerEnter(Collider other) => callBack?.Invoke();
}
```

**设计特点**：
- 使用 `UnityEvent` + `lambda` 替代传统 `if/else` 或策略模式，每个 Trigger 实例在 Inspector 中配置 `step` 值即可改变行为
- `step` 字段是 `public int`，不是 `enum`——简单直接，适合快速原型阶段
- `TipsPanel` 通过 Inspector 直连，未使用单例/事件总线解耦——当前只有一个面板场景下够用

### 配电房项目 — TipsPanel 提示面板 UI

**目标文件**: `Assets/Scripts/UI/TipsPanel.cs`

基于 TextMeshPro 的简单提示弹窗，支持动态文本 + 确认关闭：

```cs
public class TipsPanel : MonoBehaviour
{
    [SerializeField] private TextMeshProUGUI txtTips;
    [SerializeField] private Button btn_Confirm;

    public void SetTipsText(string text, bool isShowTips = true)
    {
        txtTips.text = text;
        gameObject.SetActive(isShowTips);
    }

    private void Confirm() => gameObject.SetActive(false);
}
```

**技术细节**：
- `SetTipsText` 第二个参数 `isShowTips` 默认 `true`，调用方可选择性隐藏面板（如：显示文本但先不弹出）
- 使用 `gameObject.SetActive()` 而非 Canvas/CanvasGroup 控制显隐——最简单可靠，面板不在事件响应链中时可以接受
- `btn_Confirm` 的监听在 `Start()` 里绑定，确保 Awake/OnEnable 时机安全

### 整体流程串联分析

今天三个文件的关系链：

```
玩家走入 Trigger 区域
  → OnTriggerEnter → callBack.Invoke()
    → 根据 step 调用 TipsPanel.SetTipsText()
      → 显示操作提示（"请先断电！" / "切断电源"）
```

```
GameRoot.Start()
  → Init()：按序初始化 EquipmentPanel / ScoutPanel / EnvironmentMgr / StepManager
    → StepManager.StartFlow() → EnterStep(entryStepId)
```

当前 Trigger 系统还没有和 StepManager 的步骤流转打通——Trigger 的 `step` 值是静态配置的，不随 StepManager 的 currentStepId 变化。后续需要让 Trigger 监听 `StepManager.OnStepEnter` 来动态切换 step 值。
