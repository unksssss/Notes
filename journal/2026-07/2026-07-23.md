### 配电房项目 — TaskInfo/TaskPanel 任务面板 UI 实现

**目标文件**: `Assets/Scripts/Author/Wt/UI/Task/TaskInfo.cs`, `TaskPanel.cs`

新增任务数据模型 + UI 面板，在地图上放置任务标记点并实现交互。

**TaskInfo 数据模型**（纯数据类）：
```cs
[Serializable]
public class TaskInfo
{
    public string taskLocation;       // 任务地点
    public string onFireCause;        // 起火原因
    public string taskContent;        // 任务内容
    public Transform taskOnMap;       // 任务在地图上的位置
    public Sprite onsitePhoto;        // 现场照片
    public List<EquipDefinition> neededEquip;  // 需要携带的装备
}
```

**TaskPanel 实现要点**：
- 地图标记点：用 `Resources.Load<Image>()` + `SetNativeSize()` 在运行时为地图生成图标 → 无需预制体绑定
- 交互三件套：`EventTrigger` 挂载 `PointerEnter`（放大1.1x）、`PointerExit`（还原1x）、`PointerClick`（赋值信息面板）
- `onFireCause` / `taskContent` 支持 `\\n` 通过 `Replace("\\n", "\n")` 转换为多行文本
- "进入任务"按钮：预留 `btn_EnterTask` 回调，后续切换到工具选择界面

### 配电房项目 — BackPackPanel 背包面板实现

**目标文件**: `Assets/Scripts/Author/Wt/UI/BackPackPanel.cs`

背包入口面板，从 `GameRoot.Instance.equips` 获取装备列表，实例化 `EquipmentMgr` 到 Player Camera 下。

```cs
private void OnEnable()
{
    equips.Clear();
    equips = GameRoot.Instance.equips;
    if (equipmentMgr == null)
    {
        var prefab = Resources.Load<EquipmentMgr>("Prefabs/World/Equipment");
        equipmentMgr = Instantiate(prefab, Camera.main.transform);
        equipmentMgr.Init(equips);
    }
    InitEquipmentLst();
}
```

- `EquipmentMgr` 从 Resources 加载并实例化到 `Camera.main.transform` 下（locPos=0, locRot=identity）
- `EquipmentUIItem` toggle 绑定：选中时回调 `equipmentMgr.Equip(id)` → 自动挂载装备到玩家
- ToggleGroup 确保单选

### 配电房项目 — EquipmentChoosePlus 装备选择面板三态校验

**目标文件**: `Assets/Scripts/Author/Wt/UI/EquipChoosePlus/EquipmentChoosePlus.cs`

装备选择面板核心逻辑：双向选择（全部装备 ↔ 所需装备）+ 状态图标三态校验。

**关键设计**：
```
allEquipDic  Dictionary<EquipDefinition, EquipmentItemPlus>  全部装备 O(1) 查找
equipNeededDic Dictionary<EquipDefinition, EquipNeededItem>  所需装备 O(1) 查找
neededConfigDic Dictionary<EquipDefinition, int>             需求配置 O(1) 查找
```

**ItemPlusOrMinus**（两参版本 `equip + isAdd`）：
```cs
internal void ItemPlusOrMinus(EquipDefinition equip, bool isAdd = true)
{
    if (equip == null || !equipNeededDic.TryGetValue(equip, out EquipNeededItem item)) return;
    item.Num += isAdd ? 1 : -1;
    if (item.Num == 0 && !IsEquipNeeded(equip))
    {
        equipNeededDic.Remove(equip);
        Destroy(item.gameObject);
    }
}
```
- `isAdd=true` 为添加（↑右侧箭头），`false` 为减少（←左侧箭头）
- 数量归零且为非配置装备时自动移除 UI 元素

**UpdateStatus 三态图标**：
| 状态 | 图标 | 条件 |
|------|------|------|
| ✅ Right | `sprite_Right` | 种类正确且数量正确 |
| ⚠️ Error | `sprite_Error` + 红色 | 种类正确但数量错误 |
| ❌ Wrong | `sprite_Wrong` + 红色 | 种类错误（非需求装备） |

**SetTipsText**：鼠标悬停时根据三态显示对应提示文字（种类正确数量正确 / 种类正确数量错误 / 种类错误）

**ChooseEnd**：遍历 `equipNeededDic` 逐项校验，全部通过才进入下一阶段

### 配电房项目 — EquipmentItemPlus 拖拽 Item 实现

**目标文件**: `Assets/Scripts/Author/Wt/UI/EquipChoosePlus/EquipmentItemPlus.cs`

拖拽三接口 + 目标区域检测 + 回弹动画。

```cs
public class EquipmentItemPlus : MonoBehaviour, IBeginDragHandler, IDragHandler, IEndDragHandler
{
    private bool EndAreaCheck()
    {
        return (EndArea as RectTransform).Contains(
            draggingIcon.transform as RectTransform);
    }
}
```

- **BeginDrag**：激活高亮区域 + 创建拖拽跟随图标
- **Drag**：跟随鼠标移动
- **EndDrag**：`EndAreaCheck()` 检测是否落入目标区 → 是则调用 `ItemDragEnd` 处理，否则 `DOMove` 回弹
