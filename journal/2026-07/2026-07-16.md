### 配电房项目 — EquipmentMgr：装备模型的动态管理

```cs
// Assets/Scripts/Author/Wt/Equipment/EquipmentMgr.cs
public void Init(List<EquipDefinition> equips)
{
    // 复用已有子物体而非重新 Instantiate
    for (int i = 0; i < transform.childCount; i++)
    {
        var go = transform.GetChild(i).gameObject;
        var target = equips.Find(x => x.displayName == go.name);
        if (target != null)
        {
            modelDic.Add(target.id, go);
            go.SetActive(false);
        }
        else
        {
            Destroy(go);
        }
    }
}

public void Equip(string id)
{
    foreach (var item in modelDic)
        item.Value.SetActive(false);
    if (modelDic.TryGetValue(id, out var target))
        target.SetActive(true);
}
```

**踩坑点：** 初始化逻辑从 `Instantiate` 改为「按名称匹配已有子物体」，避免每帧重复实例化。`Equip()` 先全部隐藏再激活目标，实现单一装备切换。

### 配电房项目 — BackPackPanel：背包内换装（单选）

```cs
// Assets/Scripts/Author/Wt/UI/BackPackPanel.cs
private void OnEnable()
{
    equips = GameRoot.Instance.equips;
    if (equipmentMgr == null)
    {
        var prefab = Resources.Load<EquipmentMgr>("Prefabs/World/Equipment");
        equipmentMgr = Instantiate(prefab, Camera.main.transform);
        equipmentMgr.transform.localPosition = Vector3.zero;
        equipmentMgr.Init(equips);
    }
    InitEquipmentLst();
}
```

**关键设计：**
- `EquipmentMgr` 挂载到 `Camera.main.transform`（玩家视角），`localPosition` 归零
- `ToggleGroup` 保证单选，`onValueChanged` 闭包捕获 `equip.id` 实现动态绑定

**踩坑点：** 闭包中直接引用循环变量 `equip` 会导致所有 toggle 都绑定到最后一个装备。解决方案是 `string id = equip.id;` 在闭包内捕获局部变量。

### 配电房项目 — EquipmentChoosePanel：出发前选择装备（多选）

```cs
// Assets/Scripts/Author/Wt/UI/EquipmentChoosePanel.cs
public void EquipFinish()
{
    List<EquipDefinition> resultEquipList = new List<EquipDefinition>();
    foreach (var item in equipItems)
    {
        if (!item.toggle.isOn) continue;
        var match = equips.Find(x => x.displayName == item.txt_itemName.text);
        if (match != null)
            resultEquipList.Add(match);
    }
    GameRoot.Instance.equips = resultEquipList;
    this.gameObject.SetActive(false);
}
```

**关键设计：**
- 多选模式（无 `ToggleGroup`），勾选的装备加入列表
- 确认后覆盖 `GameRoot.Instance.equips`，下游 BackPackPanel 读取该列表
- 左右翻页通过 `ScrollStepController` 控制

### 配电房项目 — ScrollStepController：可复用 ScrollView 步进滚动组件

```cs
// Assets/Scripts/Author/Wt/UI/ScrollStepController.cs
public void ScrollRight()
{
    float delta = StepToNormalized(scrollRect, stepDistance);
    _targetH = Mathf.Clamp01(_targetH + delta);
    if (!smooth)
        scrollRect.horizontalNormalizedPosition = _targetH;
}

private static float StepToNormalized(ScrollRect sr, float pixelDistance)
{
    float scrollableWidth = content.rect.width - sr.viewport.rect.width;
    if (scrollableWidth <= 0f) return 0f;
    return pixelDistance / scrollableWidth;
}
```

**关键设计：**
- `stepDistance` 以像素为单位配置，内部通过 `content宽度 - viewport宽度` 换算为 normalized 值
- 支持 `smooth` 模式（`Mathf.SmoothDamp`）和即时跳转模式
- `Update()` 中持续平滑趋近 `_targetH`，帧率无关

### 配电房项目 — 旧 EquipmentPanel 废弃

原 `EquipmentPanel` 已完全注释（`/* */`），不再启用。旧方案与 `StepManager` 紧耦合，硬编码了3件必须装备（绝缘手套、绝缘靴、绝缘操作杆），扩展性差。新方案通过 BackPackPanel + EquipmentChoosePanel 解耦，装备数据由 `EquipDefinition` ScriptableObject 配置。
