### FireSim_URP — HandleOnMouseDown 修复与重构

**目标文件**: `Assets/Scripts/Author/Wt/HandleOnMouseDown.cs`

基于 eventID 静态变量的交互状态机，完成了灭火场景全流程的代码修复与优化：

**① 错别字修正**
```diff
- ViewManager.ShowToast(null, "墙内温度过高,纯在阴燃,请灭火", ColdDownTheWall);
+ ViewManager.ShowToast(null, "墙内温度过高,存在阴燃,请灭火", ColdDownTheWall);
```

**② 重复成员访问 → 局部变量缓存**
```diff
- FireExtingction.instance.playerEquipment.Equip(EquipmentType.Extinguisher);
- FireExtingction.instance.playerEquipment.Equip(EquipmentType.Axe);
+ var eq = FireExtingction.instance.playerEquipment;
+ eq.Equip(EquipmentType.Extinguisher);
+ eq.Equip(EquipmentType.Axe);
```
4 处 `instance.playerEquipment` 链式访问统一提取为局部变量 `var eq`，减少 `instance` 属性查找开销。

**③ 交互对象 Null 安全防护**
```cs
var interactObjs = ObjectManager.Instance.InteractableObjs;
if (interactObjs != null && interactObjs.Count > 0 && interactObjs[0] != null)
    interactObjs[0].SetActive(false);
```
三层保护：列表引用 null → 列表为空 → 元素 null，防止破门动画后的对象释放 crash。

**④ 斧头动画等待协程**
```cs
private IEnumerator WaitForAnimation(Animator animator, Action callback)
{
    yield return null; // 等一帧让状态机切换
    var state = animator.GetCurrentAnimatorStateInfo(0);
    yield return new WaitForSeconds(state.length);
    callback?.Invoke();
}
```
先 yield return null 确保动画状态机完成切换，再读取当前动画时长，避免拿到上一帧状态。

### Test01 — Mesh 顶点偏移测试

**目标文件**: `Assets/Script/Test16.cs`

```cs
meshFilter = GetComponent<MeshFilter>();
vertices = meshFilter.mesh.vertices;
for (int i = 0; i < vertices.Length; i++)
{
    vertices[i] += new Vector3(0, 0, 4);
}
meshFilter.mesh.vertices = vertices;
```
读取 Mesh 的所有顶点并沿 Z 轴偏移 4 个单位后写回。注意：操作的是 `meshFilter.mesh`（共享 Mesh 实例），会**永久修改原始网格数据**，多物体共享同一 Mesh 时会有副作用。
