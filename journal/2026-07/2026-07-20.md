### C# — 扩展方法（Extension Methods）本质

**定义语法**：
```cs
public static class UnityComponentExtention
{
    // this 关键字标记第一个参数 → 给 RectTransform "追加"方法
    public static bool Contatins(this RectTransform container, RectTransform rt)
    {
        // container 就是调用者本身
        Vector3[] corners = new Vector3[4];
        container.GetWorldCorners(corners);
        // ...
    }

    // 没有 this → 普通静态方法，不能点调
    public static string HandleTextContentFormat(string content) { ... }
}
```

**本质**：纯**编译时语法糖**，运行时不存在"扩展方法"这个概念。

| 层面 | 表现 |
|:----|:-----|
| 你写的源码 | `rectTransform.Contatins(gunRect);` |
| 编译器翻译成 | `UnityComponentExtention.Contatins(rectTransform, gunRect);` |
| IL 层面 | 普通的 `call` 指令调用静态方法 |
| 运行时 | 完全不知道有扩展方法这回事 |

**关键规则**：
- 必须写在 `public static class` 里
- 第一个参数必须带 `this`，参数类型 = 被扩展的目标类型
- 不能访问目标类的 `private` 成员（本质还是外部静态方法）
- 目标类有同名方法时，优先调原方法（扩展方法优先级最低）
- `using` 没引入命名空间时找不到

**Unity 常见扩展方法示例**：
```cs
public static class GameObjectExtensions
{
    public static T GetOrAdd<T>(this GameObject go) where T : Component
        => go.GetComponent<T>() ?? go.AddComponent<T>();

    public static void Show(this GameObject go) => go.SetActive(true);
    public static void Hide(this GameObject go) => go.SetActive(false);
}

public static class TransformExtensions
{
    public static void ResetLocal(this Transform t)
    {
        t.localPosition = Vector3.zero;
        t.localRotation = Quaternion.identity;
        t.localScale = Vector3.one;
    }
}
```

### UGUI — 事件接口体系（EventSystem 三兄弟）

UGUI 的事件系统基于 **UnityEngine.EventSystems**，核心是三个拖拽接口 + 点击接口：

| 接口 | 触发时机 | 常用场景 |
|:-----|:---------|:---------|
| `IPointerClickHandler` | 点击完成（按下+抬起在同一物体） | 按钮、物品点击 |
| `IBeginDragHandler` | 开始拖拽（按下后移动超过阈值） | 拖拽起始逻辑 |
| `IDragHandler` | 拖拽中（每帧） | 跟随鼠标移动 |
| `IEndDragHandler` | 结束拖拽（松开鼠标/手指） | 放置检测、恢复位置 |

**实现方式一：继承接口（推荐给自己的脚本用）**

```cs
public class EquipmentItemPlus : MonoBehaviour,
    IBeginDragHandler, IDragHandler, IEndDragHandler
{
    private Image dragGhost;
    private Transform originParent;

    public void OnBeginDrag(PointerEventData eventData)
    {
        // 1. 克隆 icon 作为拖拽影像
        dragGhost = Instantiate(iconImage);
        dragGhost.raycastTarget = false;  // 拖拽影像不阻挡射线

        // 2. 屏幕坐标 → UI 局部坐标
        RectTransformUtility.ScreenPointToLocalPointInRectangle(
            parentRect, eventData.position,
            eventData.pressEventCamera, out Vector2 localPos);
        dragGhost.rectTransform.anchoredPosition = localPos;

        // 3. 记录原始父级（用于恢复）
        originParent = transform.parent;
        transform.SetParent(dropZone);  // 提到 DropZone 层级下
    }

    public void OnDrag(PointerEventData eventData)
    {
        // 拖拽影像跟随鼠标
        dragGhost.rectTransform.anchoredPosition += eventData.delta;
    }

    public void OnEndDrag(PointerEventData eventData)
    {
        // 检测是否落在目标区域
        if (IsInDropZone(eventData))
            OnDropSuccess();
        else
            RestorePosition();
    }
}
```

**实现方式二：EventTrigger 组件（推荐给别人/动态绑定）**

```cs
// 挂在任何 UI 物体上，运行时动态添加事件
var trigger = go.GetComponent<EventTrigger>();
if (trigger == null) trigger = go.AddComponent<EventTrigger>();

var entry = new EventTrigger.Entry();
entry.eventID = EventTriggerType.PointerClick;
entry.callback.AddListener((data) => {
    Debug.Log("点击了 " + go.name);
});
trigger.triggers.Add(entry);
```

### UGUI — EventTrigger 与接口的选择原则

| 场景 | 选什么 |
|:-----|:-------|
| 自己的脚本处理自己的点击 | `IPointerClickHandler` 接口 |
| 给别人/预制体的控件加事件 | EventTrigger |
| 运行时动态绑定事件 | EventTrigger |
| 需要 5+ 种事件，不想类太长 | EventTrigger |
| 拖拽面板 | `IBeginDragHandler` + `IDragHandler` + `IEndDragHandler` |

### UGUI — EventTrigger 不生效排查清单

1. **Image.Raycast Target** 没勾 ← 最常见
2. Canvas 上缺 **GraphicRaycaster** 组件
3. EventTrigger 组件**只挂了壳，没加 Event Type Entry**（事件列表为空）
4. 被上层 UI 遮挡（后渲染的子物体截断射线）
5. 父物体是 Button，Button 吃掉了 PointerClick 事件
   - 解法：换 PointerDown 事件，或者把 EventTrigger 放 Button 同一物体

### UGUI — PointerEventData 参数说明

`OnDrag(PointerEventData eventData)` 中 `eventData` 的关键字段：

| 字段 | 类型 | 说明 |
|:-----|:-----|:-----|
| `position` | `Vector2` | 当前屏幕坐标 |
| `delta` | `Vector2` | 上一帧到本帧的位移 |
| `pressPosition` | `Vector2` | 按下时的屏幕坐标 |
| `pointerEnter` | `GameObject` | 当前悬停的物体 |
| `pointerDrag` | `GameObject` | 当前拖拽的物体 |
| `pressEventCamera` | `Camera` | 发射射线的 Camera |
| `dragging` | `bool` | 是否正在拖拽 |

`delta` 妙用：不用记上一帧位置，直接用 `+= eventData.delta` 就是跟随鼠标。

### UGUI — Drag Ghost（拖拽影像）模式详解

**标准流程**：

```cs
OnBeginDrag → 克隆 Icon 生成影像，关闭 raycastTarget
OnDrag      → 影像.anchoredPosition += eventData.delta
OnEndDrag   → 检测落点 → 成功：销毁影像，放置物体
                         → 失败：销毁影像，物体恢复原位
```

**为什么要 `raycastTarget = false`**：
- 拖拽影像浮在鼠标上，会挡住射线检测
- 不关的话，`OnEndDrag` 时 `eventData.pointerEnter` 永远是影像本身
- 关了影像就不参与射线检测，事件才能穿透到下面的 UI

### 配电房项目 — EquipmentChoosePlus 装备选择界面三件套

今天完成了装备选择界面的三个核心脚本，采用**拖拽 + 双击 + 按钮**三种交互方式。

**Architecture**：三个类协同工作

```
EquipmentChoosePlus (Singleton 主控)
  ├─ 管理 allEquipLst（所有可选装备）
  ├─ 管理 equipNeededLst（本任务所需装备）
  └─ 管理错误装备逻辑 AddWrongItem / RemoveWrongItem

EquipmentItemPlus (拖拽源)
  └─ IBeginDragHandler + IDragHandler + IEndDragHandler
     └─ EndAreaCheck → 拖入目标区则调用 AddTargetEquipNum

EquipNeededItem (目标区条目)
  └─ IPointerClickHandler → 双击 +1
  └─ 加减按钮 → AddTargetEquipNum
  └─ Num 属性自动 clamp [0,99]，=0 时 SetInteractable(false)
```

**关键代码 — EquipmentItemPlus.EndAreaCheck**：

```cs
public void OnEndDrag(PointerEventData eventData)
{
    bool isOnEndArea = EndAreaCheck();
    // 还原拖拽icon
    img_DraggingIcon.transform.SetParent(this.transform);
    (img_DraggingIcon.transform as RectTransform).anchoredPosition = Vector2.zero;
    img_DraggingIcon.gameObject.SetActive(false);

    if (isOnEndArea)
        EquipmentChoosePlus.Instance.AddTargetEquipNum(this.tmp_name.text);
}

private bool EndAreaCheck()
{
    return (EndArea as RectTransform).Contatins(img_DraggingIcon.transform as RectTransform);
}
```

**关键代码 — EquipNeededItem 双击检测**：

```cs
private float doubleClickThreshold = 0.3f;
private float lastClickTime;

public void OnPointerClick(PointerEventData eventData)
{
    float timeSinceLastClick = Time.unscaledTime - lastClickTime;
    if (timeSinceLastClick < doubleClickThreshold && isInteractable)
    {
        OnDoubleClick();             // 双击 +1
        lastClickTime = 0;           // 防三击重复触发
    }
    lastClickTime = Time.unscaledTime;
}

private void SetInteractable(bool interactable)
{
    this.isInteractable = interactable;
    img_mask.SetActive(!interactable);     // 遮罩阻断射线
    btn_minus.interactable = interactable;
    btn_plus.interactable = interactable;
}
```

**坑点**：
- `img_DraggingIcon` 在 `OnEndDrag` 时先还原父级再关 `SetActive`，否则还原位置时因父级变更导致位置偏移
- 错误装备用独立图标（`sprite_Wrong`）标记，逻辑上通过 `Find(equip.displayName)` 匹配

### FireSim_URP — ExplosiveFireCtrl 轰燃场景时序控制器

**文件**: `Assets/Scripts/Author/Wt/ExplosiveFire/ExplosiveFireCtrl.cs`

**时序流程**：
```
StartPlay()
  → 第0个物体点燃（SetOnFireFromCenter）
  → Wait(timeToCeilingSmoke)
  → 天花板烟雾粒子渐入（FadeInSmoke）
  → Wait(timeToFadeInSmoke + timeToExplosiveFire)
  → 其余物体全部点燃（SetOtherOnFire）
  → 天花板烟雾颜色变红（color_OnFire）
```

**天花板烟雾粒子渐入**：
```cs
private IEnumerator FadeInSmoke(float duration)
{
    while (elapsed < duration)
    {
        float t = elapsed / duration;
        emission.rateOverTime = Mathf.Lerp(0, _targetRate, t);
        main.startSize = Mathf.Lerp(0.1f, _targetSize, t);
        yield return null;
    }
}
```

**熄灭反馈 → 顶棚坍塌**：
```cs
private void BeingExtinguished(float t = 0.00025f)
{
    elasped += t;
    if (elasped >= 6.9 && !hasCollapse)
    {
        CollapseCeiling();   // 触发顶棚坍塌（ObjectManager）
        hasCollapse = true;
        return;
    }
    FadeOutSmoke(t);
}
```

**排烟功能（StartEjectSmoke）**：
- 开启 `windZone` 粒子外力场，让粒子沿排烟方向运动
- 5秒等待后，渐变缩小粒子 Size 和 Emission Rate
- 排烟完成后粒子移到 `transCeilingSmokeMoveTo` 位置
- 关联 `_backdraftCtrl.SetBackdraftFire(false)` → 门后回燃火消失

## Ideas

- 扩展方法很适合做 Unity 组件的快捷操作（GetOrAdd、Show/Hide、ResetLocal）
- Drag Ghost 模式可抽象成通用组件，支持回调配置
- "编译时语法糖，运行时不存在"——这个思维模型适用于很多 C# 特性（LINQ、async/await、using 等）
- 可以给 List/Transform 等常用类型写一套项目级扩展方法库
- ExplosiveFireCtrl 的时序流程可用 ScriptableObject 配置，支持策划调参
- 后端驱动的动态字段设计（FieldValueDto[]）可推广到其他表单场景
