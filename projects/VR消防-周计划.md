# 配电柜房火灾 — 核心开发计划

**交付日：2026年8月1日** | 剩余约 8 周
**职责：场景特效 + 流程控制 + 核心交互**（UI/打分系统由他人负责）

## 四大特效

| 特效 | 状态 | 备注 |
|---|---|---|
| 回燃火 + 明火 | ✅ 已有 | OAVA Flame + NatureManufacture |
| 流动烟雾 | 🔧 调参中 | NatureManufacture Lit_Lightmap系列 |
| 线路电火花/地面电流 | ✅ 已有 | LightningBolt + GroundElectric_Random Shader |
| 阴燃 | ❌ 未实现 | 可降级为静态发光点或砍掉 |

## 流程步骤（StepManager 对接）

```
步骤01 准备     → 装备穿戴判断逻辑
步骤02 获取信息 → 场景动画触发
步骤03 侦察     → 配电柜观察交互
步骤04 断电     → 电流特效开关
步骤05 疏散     → 环境切换
步骤06 灭火     → 柜门分支 + 灭火器交互
步骤07 汇报     → （他人）
步骤08 指示     → （他人）
步骤09 隐患排除 → 排查交互逻辑
步骤10 记录     → （他人）
```

---

## 周计划

### 第1周（6/2 ~ 6/8）— 粒子特效精调 🎯

| 日期 | 任务 |
|---|---|
| 周一 | Collision模块：火球碰墙反弹/衰减 |
| 周二 | 烟雾天花板平铺 + Noise扰动 |
| 周三 | NatureManufacture烟雾材质URP适配 |
| 周四 | 火焰+烟雾+电火花三特效联动 |
| 周五 | 步骤04断电→电流消失逻辑衔接 |
| 周末 | 特效参数固化，不再反复调 |

### 第2周（6/9 ~ 6/15）— 流程控制完善

- 补完所有步骤的 StepAsset 配置（分支跳转关系）
- 步骤01装备判断逻辑（绝缘3件套检查）
- 步骤03侦察交互（观察浓烟/变形/声响）
- EnvironmentMgr 电流开关 + 灭火监听
- FirstPersonController 各步骤锁定/解锁

### 第3周（6/16 ~ 6/22）— 灭火核心交互

- 步骤06柜门判断分支（开启/关闭 → 不同灭火方式）
- 灭火器交互：拿起→瞄准→喷射→火焰缩小→ExtinguishFully
- 防火堵料/防火门放置交互
- 灭火成功监听 → 推进下一步

### 第4周（6/23 ~ 6/29）— 场景环境系统

- 配电柜场景布置（模型/碰撞体/灯光）
- 步骤05疏散时的环境状态切换
- 步骤09隐患排除：通风/测温/线路检查/接地测试
- 场景切换时的相机动画衔接

### 第5周（6/30 ~ 7/6）— 联调

- 10个步骤从头到尾跑通
- 特效与流程衔接调优
- CustomParticalExtinguish 全覆盖测试
- 步骤间跳转边界情况修复

### 第6周（7/7 ~ 7/13）— 性能优化

- Profiler 分析 CPU/GPU/内存
- 粒子上限 + 对象池
- 火焰系统性能档位（低/中/高画质）
- 场景加载优化

### 第7周（7/14 ~ 7/20）— VR适配

- VR头盔适配（Pico/Quest）
- 手柄交互代替鼠标点击
- 第一人称移动VR化
- 灭火器手柄操作

### 第8周（7/21 ~ 8/1）— 收尾

- 全流程回归测试
- Bug修复缓冲
- 打包交付

---

## 风险点

| 风险 | 应对 |
|---|---|
| VR手柄交互没经验 | 预留2周，提前调研Pico SDK |
| 阴燃做不出来 | 砍掉或用静态发光点代替 |
| 性能不达标 | 第6周用Profiler定位 |
| 美术模型还没到 | 先用Cube代替，不阻塞流程 |
