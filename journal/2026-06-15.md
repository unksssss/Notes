### 配电房项目 — ExplosiveFireCtrl：排烟系统 EjectSmoke

在轰燃控制器中新增排烟协程——Wind Zone 吹散烟雾 → 粒子渐变稀疏 → 烟雾位置迁移 → 关闭回燃火。

```cs
public void StartEjectSmoke()
{
    windZone.SetActive(true);
    if (_ceilingRoutine == null)
        _ceilingRoutine = StartCoroutine(EjectSmoke());
}

IEnumerator EjectSmoke()
{
    yield return new WaitForSeconds(5f);  // 等粒子改变方向

    float duration = 4f;
    float elapsed = 0f;
    float startSize = main.startSize.constantMax;
    float startSpeed = emission.rateOverTime.constantMax;
    float residueSize = 1f;
    float resideSpeed = startSpeed * 0.15f;

    while (elapsed < duration)
    {
        elapsed += Time.deltaTime;
        float t = elapsed / duration;
        main.startSize = Mathf.Lerp(startSize, residueSize, t);
        emission.rateOverTime = Mathf.Lerp(startSpeed, resideSpeed, t);
        yield return null;
    }

    _ps_Ceiling.transform.position = transCeilingSmokeMoveTo.position;
    _backdraftCtrl.SetBackdraftFire(false);  // 排烟完毕后关闭回燃火
}
```

**设计要点：**
- **两阶段**：先等 5s 让 Wind Zone 改变粒子方向，再 4s 渐变缩小烟雾（粒子尺寸 → 1、发射率 → 15%）
- **残留量**：残留尺寸=1、残留速率=15%——排烟后不完全消失，模拟真实排烟后的稀薄烟雾
- **烟雾位移**：排烟完成后 `_ps_Ceiling` 瞬移到 `transCeilingSmokeMoveTo` 位置（通风口上方）
- **联动回燃**：排烟完关闭门后回燃火 `SetBackdraftFire(false)`
- **防重入**：`_ceilingRoutine` 协程引用做互斥锁

### 配电房项目 — Mathf.Lerp 在 FadeOutSmoke 中的陷阱

```cs
// ❌ 错误：constantMax 恒不变 → 每帧算出一样的值，烟雾不会逐渐消失
main.startSize = Mathf.Lerp(main.startSize.constantMax, 0f, 0.00025f);

// ✅ 正确：协程里自己计时 elapsed，t 逐渐从 0 到 1
float t = elapsed / duration;
main.startSize = Mathf.Lerp(startSize, residueSize, t);
```

**根因：** `Mathf.Lerp(a, b, t)` 的第三个参数 t 是**插值比例**（0~1），不是步长。传入固定值 `0.00025f` 意味着每帧从 a 到 b 只走 0.025%，但 a 始终是 `constantMax`（不变）→ 结果永远不变。**Lerp 不自带累积**，不像 `MoveTowards` 有固定步长。渐变必须在协程里自己维护 `t = elapsed / duration`。

### 配电房项目 — WaterPosition：阵地选择触发器（新文件）

玩家走到不同位置触发不同消防操作——回燃破门 / 错误阵地 / 排烟散热。

```cs
public class WaterPosition : MonoBehaviour
{
    [SerializeField] private int posId;
    [SerializeField] private BackdraftCtrl backdraftCtrl;
    [SerializeField] private SingleCylinderHoseFromFireTruck fireTruck;
    [SerializeField] private ExplosiveFireCtrl expCtrl;

    private void OnTriggerEnter(Collider other)
    {
        fireTruck.StopDrawing();  // 停止水带铺设
        switch (posId)
        {
            case 1: Debug.Log("正确阵地"); backdraftCtrl.OpenDoor(); break;
            case 2: Debug.LogWarning("错误阵地"); backdraftCtrl.OpenDoor(); break;
            case 3: Debug.Log("正确阵地"); backdraftCtrl.OpenDoor(); break;
            case 4:  // 排烟阵地
                UIMgr.instance.txt_Ejecting.gameObject.SetActive(true);
                expCtrl.StartEjectSmoke();
                break;
        }
    }
}
```

**设计要点：**
- **触发即停止铺设**：进入任何阵地先调 `StopDrawing()` 固定水带
- **posId 策略模式**：1/2/3 触发回燃破门（2 是教学用的错误选项），4 触发排烟
- **串联三个系统**：水带铺设（SingleCylinderHose）+ 回燃（BackdraftCtrl）+ 轰燃排烟（ExplosiveFireCtrl）

### 配电房项目 — BackdraftCtrl：回燃火开关

给回燃控制器增加 `isBackdraftFire` 控制——排烟完毕后关闭门后火焰。

```cs
public bool isBackdraftFire = true;

public void OpenDoor()
{
    if (_play == null)
        _play = StartCoroutine(Play());
}

private IEnumerator Play()
{
    if (isBackdraftFire)           // 新增：只在火焰开启时播放粒子
        _ps_Backdraft.Play();
    _door.transform.DOLocalRotate(..., _doorOpenDuration).SetEase(Ease.OutBack);
    yield return new WaitForSeconds(_doorOpenDuration);
    _play = null;
}

public void SetBackdraftFire(bool isBackdraft = true)
{
    isBackdraftFire = isBackdraft;
}
```

**用途：** 排烟完毕后 `SetBackdraftFire(false)` → 玩家再走过去触发的回燃门只开门、不喷火，视觉上完成"火已灭"的叙事。

### 配电房项目 — SingleCylinderHoseFromFireTruck：StopDrawing 完善

`StopDrawing()` 从简单设 flag 改为强制完成管道铺设：

```cs
public void StopDrawing()
{
    if (_playerEndPoint != null && _isDrawing)
    {
        Vector3 finalPos = ToGround(_playerEndPoint.position);
        _bakedPathPoints.Add(finalPos);
        _lastBakedPointPosition = finalPos;
        UpdateBakedHose();       // 重建完整的 Baked Mesh
        _activePathPoints.Clear();
        _activeHose.SetPath(_activePathPoints);  // 清空 Active 段
    }
    _isDrawing = false;
}
```

**改进：** 将玩家最终位置强制加入 bakedPath → 重建完整管道 → 清空 Active 段。确保水带到阵地终点是完整的一条 Mesh，不留断裂。

### 配电房项目 — FlammableObject：SetFEI 外部绑定接口

```cs
/// <summary>
/// 为 FlammableObject 绑定 FlameEventInvoker
/// </summary>
public void SetFEI(FlameEventInvoker _flameEventInvoker)
{
    this.flameEventInvoker = _flameEventInvoker;
}
```

OAVA 框架的 `FlammableObject` 内部使用 `flameEventInvoker`，但默认是 `GetComponent` 获取自身挂载的。当 ExplosiveFireCtrl 用 `AddComponent` 动态创建 FEI 时，需要通过 `SetFEI` 把引用传回去，确保 `BeingExtinguished` 等事件能正确触发烟雾消散。

### 配电房项目 — Wind Zone 对粒子生效的配置

**问题：** Wind Zone 对粒子无效。

**解决：** Particle System 的 **External Forces** 模块必须启用，并且 Wind Zone 的 Mode 需设为 **Directional**（非 Spherical），粒子才会响应风力。
