## VFX Graph — GPUEvent 上下文

`Trigger Event Rate (Over Time)` 在 Update Particle 中不显示，是因为没有创建 GPUEvent 上下文来接收事件。

**解决方法：**
- 右键 → Create Node → **Initialize Particle (GPUEvent)** 或 **Update Particle (GPUEvent)**
- 从 Trigger Event 块拖出 Event 输出端口连线即可自动创建

GPUEvent 上下文类型：
| Context | 用途 |
|---|---|
| Initialize Particle (GPUEvent) | 事件触发时初始化新粒子 |
| Update Particle (GPUEvent) | 事件触发时更新已有粒子 |
| Output Particle (GPUEvent) 系列 | 事件触发的粒子渲染方式 |

## 流程控制可读性 — ChooseBranchByCorrectness 显式化

### 问题

```cs
// 改前：完全看不出是哪个步骤的哪个分支
StepManager.Instance.ChooseBranchByCorrectness(true);
```

步骤、分支、目标全部隐藏在 ScriptableObject 和运行时上下文中，读代码像黑盒。

### 解决方案

给 `ChooseBranchByCorrectness` 加显式 `stepId` 参数：

```cs
// StepManager.cs
public void ChooseBranchByCorrectness(string stepId, bool isCorrect)
{
    if (!_stepMap.TryGetValue(stepId, out var step)) return;
    for (int i = 0; i < step.branches.Length; i++)
    {
        if (step.branches[i].isCorrect == isCorrect)
        {
            var branch = step.branches[i];
            Debug.Log($"[StepManager] 步骤{stepId} → {(isCorrect?"正确":"错误")}分支{branch.branchId} → {branch.targetStepId}");
            OnBranchChosen?.Invoke(branch);
            EnterStep(branch.targetStepId);
            return;
        }
    }
}

// 兼容旧调用
public void ChooseBranchByCorrectness(bool isCorrect)
    => ChooseBranchByCorrectness(currentStepId, isCorrect);
```

**效果：**

```cs
// 改后：一眼看懂
StepManager.Instance.ChooseBranchByCorrectness("01", result);  // 步骤01→正确/错误分支
StepManager.Instance.ChooseBranchByCorrectness("02", true);    // 步骤02→正确分支
StepManager.Instance.ChooseBranchByCorrectness("03", true);    // 步骤03→正确分支
StepManager.Instance.ChooseBranchByCorrectness("04", true);    // 步骤04→正确分支
StepManager.Instance.ChooseBranchByCorrectness("05", true);    // 步骤05→正确分支
StepManager.Instance.ChooseBranchByCorrectness("06", true);    // 步骤06→正确分支
```

**小结：** 事件驱动架构中，隐式上下文是代码可读性的杀手。把上下文参数化（stepId 从隐式变成显式），改动极小但收益巨大。

## OAVA Flame — extinguished 标记与灭火接口

### 问题

`ResetObj()` 会将 `extinguished` 设为 `false`，导致外部无法通过 `IsExtinguished()` 检测灭火状态。

```cs
// OAVA ResetObj()
public void ResetObj()
{
    onFire = false;
    extinguished = false;  // ← 灭火后标记反而复位了
    // ...
}
```

### 解决方案

新增 `ExtinguishFully()` 方法，与 `ResetObj()` 区别仅一处：

```cs
// FlammableObject.cs — 新增
public void ExtinguishFully()
{
    onFire = false;
    extinguished = true;   // ← 唯一区别：灭火就该是 true
    // ... （其余清理逻辑与 ResetObj 一致）
    if (flameEventInvoker) flameEventInvoker.Extinguished.Invoke();
}
```

**调用链：**
```
CustomParticalExtinguish → 火焰缩到 0 → ExtinguishFully()
  → extinguished = true
  → Extinguished 事件触发
  → EnvironmentMgr 监听 → FireShelfCount-- → 全部灭完 → 进入步骤07
```

### 陷阱

OAVA 没有公开的灭火 API。`IncrementalExtinguish(pos, radius, increment)` 需要坐标+半径参数，不适合直接缩火焰的场景。`ResetObj()` 是唯一的公开清理方法但状态不对。只能自己扩展。

### EnvironmentMgr 监听设计

```cs
// 属性 setter 驱动流程：所有架子灭完自动推进
private int FireShelfCount
{
    set
    {
        fireShelfCount = value;
        if (fireShelfCount == 0)
            StepManager.Instance.ChooseBranchByCorrectness("06", true);
    }
}
```
