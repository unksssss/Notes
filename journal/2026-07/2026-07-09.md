### 配电房项目 — Obi Rope 水带 Demo 开发

**新文件**: `Assets/Scripts/RopeHoseDemo.cs` (120 行)

用 Obi Rope 实现玩家 ← 方块之间的绳子连接，滚轮控制绳长伸缩。整体架构：

```cs
void SetupRope()
{
    // ① 自动创建 ObiSolver（场景没有时）
    // ② ObiRope + ObiRopeCursor + ObiRopeLineRenderer
    // ③ Blueprint 加载后设 Attachment
    _rope.OnBlueprintLoaded += OnBlueprintLoaded;
}

void OnBlueprintLoaded(ObiActor actor, ObiActorBlueprint bp)
{
    // ④ 两端 Attachment：Cube 端 Static，Player 端 Dynamic
    SetupAttachments();
    _cursor.ChangeLength(ropeLength);
}

void Update()
{
    // ⑤ 滚轮控制绳长
    float scroll = Input.GetAxis("Mouse ScrollWheel");
    ropeLength = Mathf.Clamp(ropeLength + scroll * 2f, minLength, maxLength);
    if (_cursor != null) _cursor.ChangeLength(ropeLength);
}
```

| 组件 | 用途 |
|------|------|
| `ObiRope` | 绳子物理核心 |
| `ObiRopeCursor` | 伸缩控制 |
| `ObiRopeLineRenderer` | 绳子可视化渲染 |
| `ObiParticleAttachment` (Static) | Cube 端固定 |
| `ObiParticleAttachment` (Dynamic) | Player 端跟随 |

### 配电房项目 — Delegate 签名踩坑修复

**编译错误 CS0123**:
```cs
// ❌ 错误：签名少了一个参数
void OnBlueprintLoaded(ObiActor actor)

// ✅ 正确：ObiActor.ActorBlueprintCallback 需要两个参数
void OnBlueprintLoaded(ObiActor actor, ObiActorBlueprint bp)
```

`ObiActor.ActorBlueprintCallback` 的 delegate 签名是 `(ObiActor, ObiActorBlueprint)`，只写一个参数匹配不上编译器会报 `CS0123: No overload matches delegate`。这个坑在 Obi 的文档示例中也没有明确标注。

### 踩坑记录

1. **Blueprint Attachment 不生效场景**：粒子生成后停留在初始坐标（`(-78.09, 0, 0)`），没有拉到 Attachment Target 的位置 → 需要 Play 模式下才能看到粒子正确归位。Edit 模式下颗粒子不会移动到目标点。

2. **renderer.material 在 Edit 模式泄漏**：通过 MCP 执行 `renderer.material` 时 Unity 会警告 `Instantiating material due to calling renderer.material during edit mode. This will leak materials into the scene.` → 应改用 `renderer.sharedMaterial`。

3. **待解决**：RopeTest 场景中绳子生成后仍无法正确关联到 Cube 和 Player，Attachment 逻辑需参照 Obi 示例场景排查。

### 每日知识问答 — Day 9（7/9 周四）

| 题 | 主题 | 要点 |
|:--:|:--|:--|
| ① | **Resources.Load 泛型** | `Resources.Load<T>(path)` 编译期类型安全，无需运行时 cast；`as` 操作符无编译期检查 |
| ② | **GC 优化** | Mono 下 `foreach` 会产生装箱和 `IEnumerator` 分配；IL2CPP 下有优化（部分） |
| ③ | **CharacterController 位移** | 用 `controller.Move(velocity * deltaTime)`，传入位移量而非速度；`velocity` 是只读属性 |
| ④ | **ScriptableObject** | 多人引用同一个 SO 实例 → 共享数据非深拷贝；运行时可通过 `CreateInstance<T>()` 创建，只是不持久化 |
| ⑤ | **OVRLipSync BlendShape** | 模型 BlendShape 索引顺序必须与 OVRLipSync viseme 枚举一一对应，否则嘴唇不动或歪 |
