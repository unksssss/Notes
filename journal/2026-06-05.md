### 配电房项目 — BackdraftCtrl：回燃控制器

新建 `BackdraftCtrl`，触发回燃场景：门弹开 + 爆燃粒子喷射。

```cs
public class BackdraftCtrl : MonoBehaviour
{
    [SerializeField] private ParticleSystem _ps_Backdraft;
    [SerializeField] private GameObject _door;
    private float _doorOpenAngle = 90f;
    private float _doorOpenDuration = 0.8f;
    private Coroutine _play;

    public void OpenDoor()
    {
        if (_play == null)
            _play = StartCoroutine(Play());
    }

    private IEnumerator Play()
    {
        _ps_Backdraft.Play();
        _door.transform.DOLocalRotate(new Vector3(0, 0, _doorOpenAngle), _doorOpenDuration)
            .SetEase(Ease.OutBack);
        yield return new WaitForSeconds(_doorOpenDuration);
        _play = null;
    }
}
```

**设计要点：**
- **DOTween Ease.OutBack**：门先快速弹开（超出目标角度一点），然后回弹到目标角度——"极快到较慢"的视觉冲击，模拟爆炸冲击波推门
- **防重入**：`_play` 协程引用做互斥锁，重复调用 `OpenDoor()` 不会叠加协程
- **粒子 + 门同步触发**：`_ps_Backdraft.Play()` 和 `DOLocalRotate` 同帧启动，确保视觉联动

### 配电房项目 — DOTween Ease.OutBack 曲线特性

**需求：** 门打开要有"先极快后被弹回一下"的效果。

**选型：** `Ease.OutBack` 是 DOTween 内置缓动之一——动画值先超过目标值（Overshoot），然后回弹收敛到目标。比 `Ease.OutElastic` 少一次振荡，更像单次冲击波而不是弹簧。

| Ease 类型 | 行为 | 适用场景 |
|---|---|---|
| OutBack | 超目标→回弹→稳定，单次 | 爆炸冲击、突然打开 |
| OutElastic | 超目标→多次振荡→稳定 | 弹簧、橡皮筋 |
| OutBounce | 多次反弹，不超目标 | 弹跳球落地 |
| OutExpo | 快到慢，无回弹 | 平稳减速 |

### 配电房项目 — ExplosiveFireCtrl 烟雾渐变重构

今天对 ExplosiveFireCtrl 做了大幅重构，核心改动三块：

**1. FadeInSmoke：烟雾从无到有渐变**

```cs
private IEnumerator FadeInSmoke(float duration = 3f)
{
    var ps = _ps_Ceiling;
    var emission = ps.emission;
    var main = ps.main;
    float elapsed = 0;
    while (elapsed < duration)
    {
        elapsed += Time.deltaTime;
        float t = elapsed / duration;
        emission.rateOverTime = Mathf.Lerp(0, _targetRate, t);
        main.startSize = Mathf.Lerp(0.1f, _targetSize, t);
        yield return null;
    }
    emission.rateOverTime = _targetRate;
    main.startSize = _targetSize;
}
```

用协程逐帧 Lerp，烟雾粒子从 0 粒/秒 → 目标值、粒子尺寸从 0.1 → 目标值，产生"烟雾逐渐变浓"的过渡感，而不是突然冒烟。

**2. FadeOutSmoke：灭火时烟雾逐渐消散**

```cs
private void FadeOutSmoke()
{
    var emission = _ps_Ceiling.emission;
    var main = _ps_Ceiling.main;
    main.startSize = Mathf.Lerp(main.startSize.constantMax, 0f, 0.00025f);
    emission.rateOverTime = Mathf.Lerp(emission.rateOverTime.constantMax, 0f, 0.00025f);
    if (main.startSize.constantMax <= 0.2f)
        StopPlay();
}
```

**从 Mathf.MoveTowards 改为 Mathf.Lerp**：MoveTowards 是固定步长（0.001/帧）→ 大烟雾和小烟雾消散速度一样。Lerp 按比例（0.00025 = 每帧缩到 99.975%）→ 烟雾越大消散越快，越小消散越慢，视觉上更自然。

**3. InitFlammableObjects：为每个可燃物挂 FlameEventInvoker**

```cs
private void InitFlammableObjects()
{
    foreach (var flammableObject in _flammableObjectLst)
    {
        var FEI = flammableObject.gameObject.AddComponent<FlameEventInvoker>();
        FEI.Init();
        FEI.BeingExtinguished?.AddListener(FadeOutSmoke);
        flammableObject.SetFEI(FEI);
    }
}
```

每个燃烧物体挂上 `FlameEventInvoker`，监听 `BeingExtinguished` 事件 → 灭火时自动触发烟雾消散。Smoke 的 lifecycle 和火焰 state 解耦。

### 配电房项目 — FlameEventInvoker UnityEvent 未初始化 Bug

**问题：** `ExplosiveFireCtrl.Start()` 中 `AddComponent<FlameEventInvoker>()` 后直接 `.BeingExtinguished?.AddListener(...)` 注册成功，但实际灭火时事件没触发。

**根因：** `FlameEventInvoker` 里的所有 6 个 `UnityEvent`（`BeingExtinguished`、`Extinguished`、`BurntOut`、`Ignited`、`MaxBrightnessReached`、`BurnOutStarted`）在声明时都只声明了字段，**没有 `new UnityEvent()`**。当 OAVA 的 `FlammableObject.Start()` 里 `GetComponent<FlameEventInvoker>()` 拿到这个组件时，这些字段全是 null。

**另一个知识点：** `AddComponent<T>()` **不会调用构造函数**。Unity 用默认无参构造函数创建实例后，通过 `Awake()` / `Start()` 做初始化。所以即使在构造函数里初始化了 UnityEvent，也不可靠——Unity 在序列化/反序列化时可能重置引用类型字段。

**解决：** 添加 `Init()` 方法显式初始化所有 UnityEvent：

```cs
public void Init()
{
    BeingExtinguished = new UnityEvent();
    Extinguished = new UnityEvent();
    BurntOut = new UnityEvent();
    Ignited = new UnityEvent();
    MaxBrightnessReached = new UnityEvent();
    BurnOutStarted = new UnityEvent();
}
```

> **教训：** OAVA 是一个运行时代码生成火焰 VFX 的框架，它内部的 event 字段默认没有初始化。要在自己的控制器里用它的 event，必须先 `Init()` 再 `AddListener`。

### 配电房项目 — UnassignedReferenceException: customFireSFX

**报错栈：**
```
FlammableObject.SetupSFX → FlammableObject.SetBoxesOnFire → 
FlammableObject.TryToSetOnFireIgniteProgressIncrease → FireTrigger.FixedUpdate
UnassignedReferenceException: The variable customFireSFX of FlammableObject has not been assigned.
```

**根因：** `ExplosiveFireCtrl` 调用 `SetOnFireFromCenter()` 后，`FlammableObject` 内部在 `SetBoxesOnFire()` → `SetupSFX()` 中尝试 `Instantiate(customFireSFX, ...)`，但 `customFireSFX` 未在 Inspector 赋值。

**解决：** `FlammableObject.Start()` 中有兜底逻辑——`if (customFireSFX == null) customFireSFX = FlameEngine.instance.defaultFireSFX;`，但这个赋值发生在 `Start()` 中。如果 `SetOnFireFromCenter` 在 `Start()` 之前被调用（或 OAVA 初始化尚未完成），`customFireSFX` 就是 null。

**最终方案：** 在 Inspector 中手动给每个 FlammableObject 挂上 `customFireSFX` 预制体，不等 `Start()` 兜底。

### 配电房项目 — NM_Particles_Smoke_Lightmap_Lit Shader 调参实战

**问题：** 烟雾从玩家相机视角看不明显——明明 Render Queue=3000、Alpha=255、Blend Mode=Alpha，但近距离看像透明。

**排查过程：**
- 最初怀疑是 **Lit Shader 受光照影响**——场景光太亮把黑色烟雾"洗白"了。建议切 Unlit。
- 用户用的就是 Unlit 变体 → 不是光照问题。
- 试了改 shader 的 `_Opacity` 属性：往大了调反而有效 → 说明**不是 Alpha 通道的问题，是 shader 内部有 Opacity 参数控制整体可见度**。

**最终调参：**
- 增大 shader 的 Opacity / Opacity Power 属性
- 保持 Render Queue 3000（Transparent 队列，保证在场景物体之后渲染）
- Blend Mode: Alpha（标准透明度混合）

> **教训：** 同一个 shader 可能有多个控制透明/可见度的属性——`Alpha`、`Opacity`、`Opacity Power`、`Color.a` 都可能影响最终效果。不要只盯着 Main Color 的 Alpha 通道。

### 配电房项目 — 回燃粒子特效优化：从廉价感到真实感

**问题：** 回燃粒子特效在人物离近后看起来像贴图，很廉价。

**解决方案：**
- **缩小粒子尺寸** → 单个粒子不那么"大块"，细节更丰富
- **加大 Emission Rate** → 用大量小粒子替代少量大粒子，视觉上从"几张贴图在飞"变成"密集的火舌"

**原理：** 粒子特效的"真实感"来自细节密度。少量大粒子 = 每个粒子的贴图边界明显 → 廉价感。大量小粒子 + 高发射率 = 粒子之间互相重叠覆盖 → 边界模糊 → 自然感。

### 配电房项目 — FirstPersonController 步骤系统集成

`FirstPersonController` 今天新增了步骤系统的完整集成：

```cs
public void Init()
{
    StepManager.Instance.OnStepEnter.AddListener(PlayerStepRespond);
}

private void PlayerStepRespond(StepAsset step)
{
    switch (step.stepId)
    {
        case "01":
        case "02":
        case "04":
        case "05":
        case "07":
        case "08":
            LockCursor(false);
            break;
        case "03":
            LockCursor(false);
            cameraAni.Play();
            StartCoroutine(WaitForAnimation(cameraAni,
                () => StepManager.Instance.ChooseBranchByCorrectness("03", true)));
            break;
        case "06":
            FireEXAni.Play("FireExtinguisher");
            canSpray = true;
            LockCursor(true);
            break;
        case "09":
            LockCursor(true);
            break;
    }
}
```

**设计要点：**
- **`canSpray` 旗标**：只在步骤 06（灭火步骤）设为 true，其他步骤设为 false——防止玩家在错误步骤喷灭火器
- **步骤 03**：播放 cinema 动画后自动推进到步骤 04，用 `WaitForAnimation` 协程等待动画结束
- **步骤 07**：进入时自动停止喷射（`StopSpray`），防止状态泄露
- 大部分步骤锁定鼠标 → 只在灭火（06）和结束（09）步骤解锁，符合 VR 培训的操作节奏

### 配电房项目 — CustomParticalExtinguish 简化

今天把 `CustomParticalExtinguish.Update()` 中的 Raycast 灭火逻辑全部注释掉了，精简为两个接口：

```cs
public void StartSpray() { _particle.Play(); }
public void StopSpray()  { _particle.Stop(); _currentFlame = null; }
```

**为什么简化：** 灭火逻辑并非通过粒子碰撞实现——需要用其他方式（如 OAVA 的 extinguish 机制或触发器）。现在 `CustomParticalExtinguish` 只负责粒子开关，灭火逻辑由其他系统处理。
