### 配电房项目 — ExplosiveFireCtrl：轰然场景多物体连锁燃烧

新增 `ExplosiveFireCtrl` 控制轰然场景——从配电柜燃烧蔓延到整个空间的火焰效果。

```cs
public class ExplosiveFireCtrl : MonoBehaviour
{
    [SerializeField] private ParticleSystem _ps_Ceiling;                    // 天花板烟雾
    [SerializeField] List<FlammableObject> _flammableObjectLst;             // 第0个最先烧
    private Material _ceilingMat;

    private void Start()
    {
        _ceilingMat = _ps_Ceiling.GetComponent<ParticleSystemRenderer>().material;
        _ceilingMat.SetColor("_Emission_Color", original_emissionColor);    // 初始无色
    }

    private IEnumerator Play()
    {
        _flammableObjectLst[0].SetOnFireFromCenter();    // 1. 第一物体燃烧
        yield return new WaitForSeconds(3f);               // 2. 等3秒火焰蔓延
        _ps_Ceiling.Play();                                // 3. 天花板烟雾起
        yield return new WaitForSeconds(5f);               // 4. 等5秒轰然发生
        SetOtherOnFire(_flammableObjectLst);               // 5. 其余物体全部引燃
        SetPs_Ceiling();                                   // 6. 天花板变红
    }
}
```

**设计要点：**

- **Coroutine 时间链**：用 `WaitForSeconds` 精确控制轰然的节奏感——先局部起火 → 烟雾升腾 → 轰然全场燃烧，比一次性全点燃更真实
- **通过 Material.SetColor 动态改 Emission**：天花板烟雾从透明暗色 → 轰然爆发后的暗红色，`_Emission_Color` 是 NatureManufacture `NM_Particles_Smoke_Lightmap_Lit.shader` 暴露的 HDR 属性
- **第0个特殊化**：List 第0个物体是起火点，先单独点燃；其余物体通过 `SetOtherOnFire` 批量处理

### 配电房项目 — FlammableObject 猛烈燃烧参数配置

要让物体**一开始就火势猛烈**，需要在 `SetOnFireFromCenter()` 前调大火焰参数：

```cs
private void SetOtherOnFire(List<FlammableObject> flammableObjects)
{
    for (var i = 1; i < flammableObjects.Count; i++)
    {
        var f = flammableObjects[i];
        f.flameParticleSize = 2f;        // 火焰粒子大（默认1）
        f.flameVFXMultiplier = 2f;       // 粒子数量多（默认1）
        f.flameLivelinessSpeed = 2f;     // 火焰跳动剧烈（默认1）
        f.ignitionTime = 0.1f;           // 点燃时间极短（默认更长）
        f.SetOnFireFromCenter();
    }
}
```

**四个参数的含义：**
| 参数 | 默认值 | 轰然取值 | 效果 |
|---|---|---|---|
| `flameParticleSize` | 1 | 2 | 单个火焰粒子更大，视觉上火焰"粗壮" |
| `flameVFXMultiplier` | 1 | 2 | 粒子发射量翻倍，火焰密度高 |
| `flameLivelinessSpeed` | 1 | 2 | 噪声动画速度翻倍，火焰跳动剧烈 |
| `ignitionTime` | 默认 | 0.1 | 几乎瞬间点燃，无渐入过渡 |

### 配电房项目 — Emission Color HDR 强度踩坑

**问题：** 用 `((Color)new Color32(191, 14, 0, 0)) * 7f` 设置 emission color，Inspector 面板显示 Intensity 为 2.8073 而不是期望的 7。

**根因：** Material Inspector 里的 Emission "Intensity" 不等于颜色乘法系数。它是从 HDR Color 的 RGB 分量经过 Gamma/Linear 转换后重新计算的——和直接 `* 7f` 不是线性关系。

**解决：** 直接用 HDR Color 构造器写目标值，不依赖乘法换算：
```cs
// ❌ 不可靠——Inspector 显示和乘法系数不对应
private readonly Color _emissionColor = ((Color)new Color32(191, 14, 0, 0)) * 7f;

// ✅ 直接给 HDR 值——所见即所得
private readonly Color _emissionColoronFire1 = new Color(167.68f, 12.32f, 0f);
```

> **教训：** Color32 的值域是 0-255（byte），强转 Color 后归一化到 0-1，再乘系数进入 HDR 域。此时 Inspector 显示的 Intensity 是重新计算的值，不直接等于乘数。要想精确控制 HDR emission，直接用 `new Color(r, g, b, a)` 构造器填入目标值。

### 配电房项目 — StepManager：添加显式 stepId 重载

给 `ChooseBranchByCorrectness` 增加了带 `stepId` 参数的显式版本：

```cs
/// <summary>
/// 在指定步骤中选择正确/错误分支 — 显式指定步骤，可读性优先
/// </summary>
public void ChooseBranchByCorrectness(string stepId, bool isCorrect)
{
    if (!_stepMap.TryGetValue(stepId, out var step))
    {
        Debug.LogError($"步骤 {stepId} 不存在");
        return;
    }
    for (int i = 0; i < step.branches.Length; i++)
    {
        if (step.branches[i].isCorrect == isCorrect)
        {
            var branch = step.branches[i];
            OnBranchChosen?.Invoke(branch);
            EnterStep(branch.targetStepId);
            return;
        }
    }
    Debug.LogError($"步骤 {stepId} 没有 isCorrect={isCorrect} 的分支");
}

/// <summary>
/// 在当前步骤中选择正确/错误分支（兼容旧调用）
/// </summary>
public void ChooseBranchByCorrectness(bool isCorrect)
{
    ChooseBranchByCorrectness(currentStepId, isCorrect);
}
```

**设计要点：**
- 显式传 `stepId` → 调用方明确知道在操作哪个步骤，降低"当前步骤不对"的排查成本
- 旧版单参数版本内部转发到新版 → 向后兼容，不需要改现有调用
- 日志输出当前步骤和分支信息，方便调试

### 配电房项目 — ParticleSystem StartSpeed 与粒子密度

**现象：** 增大 `Start Speed` 后粒子变得稀疏。

**原因：** Start Speed 提高 → 每个粒子在相同时间内飞得更远 → 同一时刻视野内的粒子间距变大 → 视觉上变稀疏。发射速率没变，只是粒子分布更散了。

**补偿方案：**
- 增大 **Rate over Time**（发射速率）→ 单位时间喷更多粒子，填补间距
- 增大 **Start Lifetime** → 粒子存活更久，视野内粒子数增多
- 配合 **Bursts** 在发射瞬间补偿密度
