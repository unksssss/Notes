# 双 Mesh 多折点水带实现解析

本文解析当前优化后的水带方案：

```text
BakedHoseMesh：已经固定的长水带，低频重建
ActiveHoseMesh：最后一小段实时水带，每帧重建
```

相关脚本：

- `SingleCylinderHoseFromFireTruck.cs`
- `MultiPointBentCylinder.cs`

核心目标是：保留“水带末端实时跟随 Player”的效果，同时避免路径变长后每帧重建整条 Mesh。

---

## 一、整体设计

旧版是：

```text
消防车起点 + 所有 bendPoint + Player 当前点 -> 每帧重建整条 Mesh
```

当玩家绕房屋一圈后，路径点变多，每帧重建的顶点、三角面和曲线采样都会变多，所以帧率会下降。

新版拆成两个 Mesh：

```text
BakedHoseMesh:
消防车起点 -> bendPoint1 -> bendPoint2 -> bendPoint3 ...

ActiveHoseMesh:
最后一个固定 bendPoint -> Player 当前点
```

也就是说：

- 长水带已经固定的部分，只在玩家每走满 `_bendPointInterval` 时更新一次。
- 短水带活动部分，每帧更新，但它始终只有 2 个路径点，开销很小。

最终视觉上仍然是一条连续水带，只是实现上由两个 Mesh 拼接。

---

## 二、脚本职责

### 1. SingleCylinderHoseFromFireTruck

这个脚本挂在消防车物体上。

它负责：

- 监听 Player 进入消防车 Trigger。
- 读取 Player 的 `lineRenderEndPoint`。
- 维护固定路径点 `_bakedPathPoints`。
- 每走 3m 记录一个新的 bendPoint。
- 创建两个运行时水带 Mesh：
  - `Runtime Baked Hose Mesh`
  - `Runtime Active Hose Mesh`
- 低频更新 Baked Mesh。
- 每帧更新 Active Mesh。

它不直接生成顶点和三角面。

### 2. MultiPointBentCylinder

这个脚本负责根据路径点生成弯曲圆柱 Mesh。

它负责：

- 接收路径点列表。
- 使用 Catmull-Rom 曲线平滑路径。
- 沿路径生成圆形截面。
- 连接截面形成圆柱侧面。
- 可选封口。
- 可选生成双面网格。

`BakedHoseMesh` 和 `ActiveHoseMesh` 都使用同一个 `MultiPointBentCylinder` 组件。

---

## 三、SingleCylinderHoseFromFireTruck 代码解析

### 1. 关键字段

```csharp
[SerializeField] private Transform _startPoint;
```

消防车出水口，作为整条水带的起点。

```csharp
[SerializeField] private Material _hoseMaterial;
[SerializeField] private float _pipeRadius = 0.05f;
[SerializeField] private float _groundY = 0f;
[SerializeField] private float _bendPointInterval = 3f;
```

这些是水带材质、半径、贴地高度和 bendPoint 生成间隔。

`_bendPointInterval = 3f` 表示玩家每走 3 米，固定一个新的 bendPoint。

```csharp
private readonly List<Vector3> _bakedPathPoints = new List<Vector3>();
```

已经固定下来的路径点。

它的结构是：

```text
消防车起点 -> bendPoint1 -> bendPoint2 -> bendPoint3 ...
```

这部分会传给 `BakedHoseMesh`。

```csharp
private readonly List<Vector3> _activePathPoints = new List<Vector3>();
```

活动路径点。

它每帧只包含：

```text
最后一个固定点 -> Player 当前点
```

这部分会传给 `ActiveHoseMesh`。

```csharp
private MultiPointBentCylinder _bakedHose;
private MultiPointBentCylinder _activeHose;
```

两个水带 Mesh 组件。

`_bakedHose` 是长的固定部分。

`_activeHose` 是短的实时部分。

```csharp
private Vector3 _lastBakedPointPosition;
```

记录上一次固定 bendPoint 的位置。

玩家当前位置和它距离超过 `_bendPointInterval` 时，就新增一个固定点。

---

### 2. OnTriggerEnter：开始生成水带

```csharp
private void OnTriggerEnter(Collider other)
{
    if (!other.CompareTag("Player")) return;
```

只有 Player 进入 Trigger 才开始。

```csharp
FirstPersonController player = other.GetComponent<FirstPersonController>();
if (_startPoint == null || player == null || player.lineRenderEndPoint == null) return;
```

检查消防车出水口、Player 控制器和 Player 末端点。

```csharp
_playerEndPoint = player.lineRenderEndPoint;
_isDrawing = true;
```

保存 Player 末端，并进入绘制状态。

```csharp
CreateOrResetHoses();
```

创建或复用两个运行时 Mesh。

```csharp
Vector3 start = ToGround(_startPoint.position);
_bakedPathPoints.Clear();
_bakedPathPoints.Add(start);
_lastBakedPointPosition = start;
```

初始化固定路径。

刚开始只有消防车起点。

```csharp
UpdateBakedHose();
UpdateActiveHose();
```

刷新两个 Mesh。

此时 Baked 只有一个点，所以没有实际网格。

Active 会显示：

```text
消防车起点 -> Player 当前点
```

---

### 3. Update：低频更新 Baked，高频更新 Active

```csharp
if (!_isDrawing || _playerEndPoint == null || _activeHose == null) return;
```

没有开始绘制或引用缺失时直接返回。

```csharp
Vector3 playerPos = ToGround(_playerEndPoint.position);
```

获取 Player 当前终点，并投影到地面高度。

```csharp
if (Vector3.Distance(_lastBakedPointPosition, playerPos) >= _bendPointInterval)
{
    _bakedPathPoints.Add(playerPos);
    _lastBakedPointPosition = playerPos;
    UpdateBakedHose();
}
```

这是性能优化的关键。

只有当玩家离上一个固定点超过 3m 时：

1. 把当前位置加入 `_bakedPathPoints`。
2. 这个点成为新的 bendPoint。
3. 重建一次 `BakedHoseMesh`。

平时玩家移动但没走满 3m 时，不重建长水带。

```csharp
UpdateActiveHose();
```

Active Mesh 每帧更新。

但它只包含两个点：

```text
最后一个固定点 -> Player 当前点
```

所以每帧重建的 Mesh 很短，性能压力很小。

---

### 4. CreateOrResetHoses：创建两个 Mesh 对象

```csharp
if (_bakedHose == null)
    _bakedHose = CreateHoseObject("Runtime Baked Hose Mesh");

if (_activeHose == null)
    _activeHose = CreateHoseObject("Runtime Active Hose Mesh");
```

运行时创建两个对象：

- `Runtime Baked Hose Mesh`
- `Runtime Active Hose Mesh`

它们都挂在消防车物体下面。

```csharp
ConfigureHose(_bakedHose);
ConfigureHose(_activeHose);
```

统一设置半径和材质。

---

### 5. UpdateBakedHose

```csharp
private void UpdateBakedHose()
{
    _bakedHose.SetPath(_bakedPathPoints);
}
```

把固定路径点传给 Baked Mesh。

当 `_bakedPathPoints` 只有一个点时，`MultiPointBentCylinder` 会清空 Mesh 并返回。

当有两个或更多固定点时，Baked Mesh 才会生成实际水带。

---

### 6. UpdateActiveHose

```csharp
_activePathPoints.Clear();

Vector3 activeStart = _bakedPathPoints[_bakedPathPoints.Count - 1];
Vector3 activeEnd = ToGround(_playerEndPoint.position);
```

Active 段的起点是最后一个固定点。

Active 段的终点是 Player 当前点。

```csharp
_activePathPoints.Add(activeStart);

if (Vector3.Distance(activeStart, activeEnd) > 0.001f)
    _activePathPoints.Add(activeEnd);
```

如果起点和终点不重合，就形成一段短水带。

如果两点几乎重合，只传一个点，`MultiPointBentCylinder` 会清空 Mesh。

```csharp
_activeHose.SetPath(_activePathPoints);
```

每帧只更新这条短路径。

## 四、MultiPointBentCylinder 代码解析

`MultiPointBentCylinder` 是通用网格生成器。

它不关心路径点来自 Baked 还是 Active，只要收到一组点，就生成一条弯曲圆柱。

### 1. 组件要求

```csharp
[RequireComponent(typeof(MeshFilter), typeof(MeshRenderer))]
```

这个组件需要：

- `MeshFilter`：保存 Mesh。
- `MeshRenderer`：渲染 Mesh。

---

### 2. 网格参数

```csharp
[SerializeField, Min(0.001f)] private float _radius = 0.05f;
[SerializeField, Min(3)] private int _radialSegments = 16;
[SerializeField, Min(1)] private int _samplesPerPathSegment = 8;
[SerializeField] private bool _capEnds = true;
[SerializeField] private bool _doubleSided = true;
```

含义：

- `_radius`：管道半径。
- `_radialSegments`：圆形截面的分段数。
- `_samplesPerPathSegment`：每两个路径点之间采样多少点。
- `_capEnds`：是否封住两端。
- `_doubleSided`：是否生成双面三角形。

---

### 3. SetPath

```csharp
public void SetPath(IList<Vector3> worldPathPoints)
{
    _worldPathPoints.Clear();

    if (worldPathPoints != null)
    {
        for (int i = 0; i < worldPathPoints.Count; i++)
            _worldPathPoints.Add(worldPathPoints[i]);
    }

    RebuildMesh();
}
```

外部传入世界坐标路径点。

脚本复制这些点，然后调用 `RebuildMesh()`。

Baked 调用频率低。

Active 每帧调用，但路径很短。

---

### 4. RebuildMesh

```csharp
CreateMesh();
_mesh.Clear();

if (_worldPathPoints.Count < 2)
    return;
```

先确保 Mesh 存在并清空旧数据。

如果路径点少于 2 个，无法形成圆柱，直接返回。

```csharp
SamplePath();
```

使用 Catmull-Rom 曲线对路径采样。

```csharp
int ringCount = _sampledLocalPoints.Count;
int sideVertexCount = ringCount * _radialSegments;
int capVertexCount = _capEnds ? 2 : 0;
```

每个采样点对应一圈圆形截面。

顶点数量约等于：

```text
采样点数量 * 圆截面分段数
```

---

### 5. 沿路径生成圆形截面

```csharp
Vector3 center = _sampledLocalPoints[i];
Vector3 tangent = GetTangent(i);
```

`center` 是当前圆环中心。

`tangent` 是路径方向。

```csharp
Vector3 xAxis = Vector3.Cross(Vector3.up, tangent);
if (xAxis.sqrMagnitude < 0.000001f)
    xAxis = Vector3.Cross(Vector3.right, tangent);
xAxis.Normalize();

Vector3 yAxis = Vector3.Cross(tangent, xAxis).normalized;
```

构造当前圆截面的两个轴。

这两个轴都垂直于路径方向。

```csharp
float angle = Mathf.PI * 2f * j / _radialSegments;
Vector3 radial = Mathf.Cos(angle) * xAxis + Mathf.Sin(angle) * yAxis;
vertices[index] = center + radial * _radius;
```

沿圆周生成顶点。

---

### 6. 连接相邻圆环

相邻两圈圆之间形成很多四边形。

每个四边形拆成两个三角形：

```text
a ---- b
|      |
c ---- d
```

代码生成：

```csharp
triangles[tri++] = a;
triangles[tri++] = b;
triangles[tri++] = c;

triangles[tri++] = b;
triangles[tri++] = d;
triangles[tri++] = c;
```

这样所有圆环被连接成圆柱侧面。

---

### 7. 双面网格

```csharp
if (_doubleSided)
{
    for (int i = 0; i < frontTriangleIndexCount; i += 3)
    {
        triangles[tri++] = triangles[i];
        triangles[tri++] = triangles[i + 2];
        triangles[tri++] = triangles[i + 1];
    }
}
```

Unity 默认背面不渲染。

这里复制一份反向三角形，让水带正反两面都能看到。

---

## 五、路径平滑：Catmull-Rom 曲线

`MultiPointBentCylinder` 使用 Catmull-Rom 曲线平滑路径。

优点：

- 曲线会经过路径点。
- 不需要额外控制点。
- 适合玩家移动轨迹。

每段曲线使用 4 个点：

```text
p0, p1, p2, p3
```

当前曲线段从 `p1` 到 `p2`。

`p0` 和 `p3` 用于决定进入和离开的方向。

核心公式：

```csharp
return 0.5f * (
    2f * p1 +
    (-p0 + p2) * t +
    (2f * p0 - 5f * p1 + 4f * p2 - p3) * t2 +
    (-p0 + 3f * p1 - 3f * p2 + p3) * t3);
```

## 六、为什么性能提升明显

旧版每帧重建：

```text
整条路径全部采样
整条水带全部顶点重算
整条水带全部三角形重算
```

玩家走得越远，路径越长，每帧开销越大。

新版每帧只重建：

```text
最后一个固定点 -> Player 当前点
```

也就是 Active Mesh。

它通常只有 2 个路径点，即使每帧重建也很轻。

Baked Mesh 虽然会越来越长，但它只在玩家每走满 3m 时重建一次，不再每帧重建。

所以路径变长后，普通帧的 Mesh 重建成本基本不增长。

---

## 七、当前方案的取舍

### 优点

- 末端仍然实时跟随 Player。
- 长路径不会导致每帧开销持续增大。
- 实现比真正的单 Mesh 增量更新简单很多。
- Hierarchy 里只有两个水带 Mesh 对象，不会产生大量段对象。
- Baked 和 Active 都复用同一个网格生成器。

### 取舍

视觉上是一条水带，但技术上由两个 Mesh 拼接。

在材质、半径、贴地高度一致的情况下，拼接处通常不明显。

如果以后需要完全无缝，可以进一步让 Active 的起点截面方向继承 Baked 末端方向，但当前贴地水带场景一般够用。

---

## 八、Unity 使用方式

1. 在消防车物体上挂 `SingleCylinderHoseFromFireTruck`。
2. 消防车物体需要有 Collider，并勾选 `Is Trigger`。
3. Player 的 Tag 设置为 `Player`。
4. Player 上有 `FirstPersonController`。
5. 设置好 `FirstPersonController.lineRenderEndPoint`。
6. 在 `SingleCylinderHoseFromFireTruck` 中拖入：
   - `_startPoint`：消防车出水口。
   - `_hoseMaterial`：水带材质。
7. 根据需要调整：
   - `_pipeRadius`：水带粗细。
   - `_groundY`：贴地高度。
   - `_bendPointInterval`：默认 3m。

运行后：

```text
玩家进入 Trigger
创建 BakedHoseMesh 和 ActiveHoseMesh
ActiveHoseMesh 每帧跟随 Player
玩家每走 3m，当前位置写入 BakedHoseMesh
ActiveHoseMesh 从新的固定点继续延伸
```
