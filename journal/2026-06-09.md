# 2026-06-09 星期一

## Records

### 配电房项目 — 水带铺设系统完整重构

从 `BezierHose`（多段独立圆柱体）演进到**单 Mesh 程序化弯曲管道**，分三层实现：

#### 三层架构

| 类 | 职责 |
|---|---|
| `ThreePointBentCylinder` | 三点贝塞尔弯曲圆柱体基础实现 |
| `MultiPointBentCylinder` | 多点 Catmull-Rom 样条 → 单个弯曲 Mesh |
| `SingleCylinderHoseFromFireTruck` | 消防车水带控制器：双 Mesh 分层优化 |

#### MultiPointBentCylinder — 程序化弯曲管道

核心技术点：

- **Catmull-Rom 样条插值**：通过所有控制点的平滑曲线，比贝塞尔更自然
- **环形截面顶点生成**：每个采样点 `(radialSegments=16)` 生成一圈顶点
- **切线计算**：中心差分法，首尾端点用单侧差分
- **双面渲染**：VR 里管道从各个角度都能看到
- **封口可选**：`_capEnds` 控制两端是否加圆盖

```cs
// Catmull-Rom 核心公式
return 0.5f * (
    2f * p1 +
    (-p0 + p2) * t +
    (2f * p0 - 5f * p1 + 4f * p2 - p3) * t2 +
    (-p0 + 3f * p1 - 3f * p2 + p3) * t3);
```

#### SingleCylinderHoseFromFireTruck — 性能关键优化

**核心思路：路径变长 ≠ 全部重建**

```
BakedHoseMesh    → 长段已固定水带，仅在走满 _bendPointInterval 时重建
ActiveHoseMesh   → 末端短段，跟随玩家每帧实时更新
```

| | Baked Mesh | Active Mesh |
|---|---|---|
| 更新频率 | 每 3 米一次 | 每帧 |
| 内容 | 消防车起点 → 上一个固定点 | 上一固定点 → 玩家当前位置 |
| 顶点数 | 十几米 × 插值 | 最多 3 米 × 插值 |

**避免**：路径变长后每帧重建整条 Mesh 的性能灾难。
