### UnityYAMLMerge — SVN 场景冲突合并配置

Unity 内置的 YAML 三路合并工具，解决多人编辑同一场景的冲突合并。

**前置设置**：
```
Project Settings → Editor
├── Version Control Mode: Visible Meta Files
└── Asset Serialization: Force Text
```

**路径**：
```
C:\Program Files\Unity\Hub\Editor\2022.3.61f1c1\Editor\Data\Tools\UnityYAMLMerge.exe
```

**TorpoiseSVN 配置**（只设合并，diff 用 SVN 自带）：
```
Settings → Diff Viewer → Advanced → .unity →
  "C:\...\UnityYAMLMerge.exe" merge -p %base %theirs %mine %merged
```

| 场景 | 参数 |
|---|---|
| Diff（比较） | `diff %mine %base` |
| Merge（合并） | `merge -p %base %theirs %mine %merged` |

**合并原理**：按 YAML `--- !u!` 分隔符，将不同 GameObject 独立合并。同个 GameObject 两人都改仍需手动修复。

**测试方法**：
```cmd
UnityYAMLMerge.exe diff "文件1.unity" "文件2.unity"
fc 文件1.unity 文件2.unity   # 验证是否真有差异
```

### FlammableObject 灭火连锁 bug 排查

**现象**：灭一个物体后，其他物体也被同时灭掉。

**根因**：`ExplosiveFireCtrl.FadeOutSmoke()` → `StopPlay()` 遍历 `_flammableObjectLst` 全部 `ExtinguishFully()`。

```cs
// FadeOutSmoke 里的问题代码
if (main.startSize.constantMax <= 0.2f)
    StopPlay();  // ← 杀了全部可燃物！
```

**修复**：`StopPlay()` 拆出，天花板烟雾归零时只停粒子，不杀其他物体。

### ShaderGraph 域重载报错

`GraphObject.OnBeforeSerialize` 报错：ShaderGraph 在域重载时内部时序问题。修法：Reimport .shadergraph 或删 Library/ShaderCache。

### 每日知识问答 — Day 5（5/5 ✅）

**Q1 — UnityYAMLMerge 前提**：[A] Force Text 序列化 ✅
> 不开 Force Text，.unity 是二进制文件，任何 diff/merge 工具都无能为力。这是前置条件。

**Q2 — 灭火连锁 bug**：[A] FadeOutSmoke → StopPlay 遍历 ExtinguishFully ✅
> 链路：BeingExtinguished → FadeOutSmoke → StopPlay() → 所有 _flammableObjectLst 全部 ExtinguishFully。天花板烟雾归零不该杀其他物体。

**Q3 — materials vs sharedMaterials**：[A] materials 每次 new 拷贝 ✅
> `smr.materials` 每次都创建新数组，别放 Update。`sharedMaterials` 返回原始引用，改了影响所有实例。

**Q4 — diff vs merge 参数**：[B] merge -p 3路合并 ✅
> diff 只需 2 参数（%mine %base），merge 需要 4 参数 + -p。配 SVN 时分开设以免参数错乱。

**Q5 — ShaderGraph OnBeforeSerialize**：[A] 域重载时内部时序问题 ✅
> 域重载期间 ShaderGraph 抢跑导致。Reimport .shadergraph 或删 Library/ShaderCache。频繁 Play/Stop 容易触发。
